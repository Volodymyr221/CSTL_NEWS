# 🏗️ ARCHITECT PLAN: CSTL NEWS MVP (v1.0)

Цей документ є головною інструкцією для розробки. Всі зміни мають відповідати описаним нижче модулям.

## 1. Система ідентифікації та зворотного зв'язку
- **Anonymous ID:** При першому запуску генерувати `deviceId` (UUID) та зберігати в `localStorage`.
- **Внутрішня форма:** Замінити `mailto:` на `fetch`-запит до API (Web3Forms/EmailJS). 
- **Мета:** Відправка новин/відгуків без виходу з PWA. Передавати `deviceId` разом із даними форми.

## 2. Модуль "Розумний Транспорт" (Автобуси)
- **Парсинг даних:** Реалізувати логіку оновлення розкладу з зовнішніх джерел (автостанції).
- **Tracking System:** Функція "Запланувати поїздку". При натисканні — підписка на конкретний `trip_id`.
- **Emergency Notifications:** Якщо в отриманих даних рейс позначено як "Скасовано", а користувач його відстежує — негайно вивести Push-сповіщення (через Service Worker).

## 3. Модуль "Енерго-Варта" (Світло)
- **Персоналізація:** Вибір району/вулиці або "Черги" користувачем. Збереження вибору.
- **Dynamic Status:** Парсинг актуальних графіків (Волиньобленерго).
- **UI:** Візуальний статус (Є світло/Немає/Можливе відключення) + таймер зворотного відліку до зміни стану.

## 4. Погода та Інтерфейс
- **Smart Weather:** Віджет у хедері. Пріоритет — геолокація користувача (через `navigator.geolocation`). Якщо доступ заблоковано — дефолт на координати Олики.
- **Animations:** Додати легкі CSS-переходи (0.2s-0.3s opacity/transform) між вкладками та при відкритті модальних вікон статей.
- **History API:** Перехоплення кнопки "Назад" у браузері/смартфоні для закриття модальних вікон без виходу з додатка.

## 5. Технічні вимоги (Stark Standards)
- **Vanilla JS Only:** Жодних важких фреймворків. Максимальна швидкість завантаження.
- **PWA Focus:** Коректна робота в офлайні (кешування останніх успішно завантажених даних).
- **Auto-Sync:** Автоматичне оновлення даних (fetch) при переході пристрою з офлайну в онлайн.
