# /deploy — Деплой (публікація на сервер)

Виконай всі кроки по порядку. Не пропускати жодного.

## Крок 1 — Час
Запусти `date` в терміналі. Запам'ятай результат — він потрібен для CACHE_NAME (назви кешу).

## Крок 2 — CACHE_NAME (назва кешу — версія збережених файлів)
Відкрий `sw.js` і зміни `CACHE_NAME` на формат `cstl-YYYYMMDD-HHMM`.
Приклад: `cstl-20260407-1430`.
НЕ вигадувати час — використовувати результат з кроку 1.

## Крок 3 — Перевірка синтаксису (правильності коду)
Запусти `node --check` для кожного зміненого `.js` файлу.
Якщо є помилка — СТОП. Спочатку виправити, тільки тоді далі.

## Крок 4 — Оновлення документації

### ЗАВЖДИ — `_ai-tools/SESSION_STATE.md`
- **Оновлено:** сьогоднішня дата
- **Що зроблено:** коротко що змінювалось в цій сесії
- **Відкриті задачі:** оновити якщо щось закрито або з'явилось нове

### Якщо змінювалась структура файлів — `CLAUDE.md`
- Таблиця "Файлова структура" — додати новий файл

### Якщо змінювалась архітектура — `docs/ARCHITECTURE.md`
- Оновити діаграми і описи

## Крок 5 — Git push (відправка змін на сервер)
```
git add .
git commit -m "опис що зроблено"
git push -u origin main
```

## Крок 6 — Перевірка CI (системи автоматичного деплою)
Зайди в GitHub → Actions і переконайся що деплой пройшов (зелений кружечок).
Якщо CI не спрацював через ~3 хвилини:
```
git commit --allow-empty -m "ci: retrigger" && git push origin main
```
