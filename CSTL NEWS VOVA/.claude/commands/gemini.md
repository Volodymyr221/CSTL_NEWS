# /gemini — Запит другої думки від Gemini (ручна вставка в aistudio.google.com)

**Аргумент:** питання до Gemini (необов'язково). Приклади:
- `/gemini перевір мої зміни перед деплоєм`
- `/gemini я не знаю як краще зробити RSS агрегатор, запропонуй архітектуру`
- `/gemini` (без аргументу — тоді спитати Вову що він хоче від Gemini)

## Як це працює (концепція)

Це **не автоматичний MCP**. Це ручний workflow (ручний процес):
1. Claude збирає повний контекст проекту + git diff + питання Вови
2. Виводить ОДНИМ код-блоком (щоб з'явилась кнопка Copy у claude.ai/code)
3. Вова копіює → відкриває https://aistudio.google.com → вставляє → читає відповідь Gemini
4. Вова повертає відповідь Gemini в чат → Claude реагує (погоджується або аргументує)

Інфраструктура нульова. Ключі ніде не засвічуються. Gemini Pro має 1M+ контекст — вміщає весь CSTL NEWS.

## Крок 1 — Отримати питання від Вови

Якщо після `/gemini` є текст — це питання, іди до кроку 2.

Якщо аргумент порожній — запитати:
> **Що ти хочеш від Gemini?** (наприклад: "перевір перед деплоєм", "запропонуй архітектуру для X", "знайди баги в моєму підході до Y")

Чекати відповідь Вови перед переходом до кроку 2.

## Крок 2 — Зібрати контекст

Виконати по порядку:

1. **Прочитати `CLAUDE.md`** (правила проекту — повний файл)
2. **Прочитати `_ai-tools/SESSION_STATE.md`** (поточний стан сесії)
3. **`git branch --show-current`** — дізнатись поточну гілку (branch — гілка — окрема лінія розробки в git)
4. **`git diff main...HEAD`** — зміни від main (головна гілка) до HEAD (поточна позиція)
   - Якщо diff (різниця) порожній — пропустити, не включати в промпт
5. **`git diff main...HEAD --name-only`** — список змінених файлів
   - Для кожного файлу зі списку — **прочитати повністю** (Read tool). Винятки:
     - `bundle.js` — НЕ читати (згенерований, не потрібен Gemini)
     - `package-lock.json` — НЕ читати (занадто великий, не потрібен)
     - Бінарні файли (зображення, шрифти) — НЕ читати

## Крок 3 — Сформувати і вивести промпт (інструкцію для Gemini)

Проаналізуй контекст і дай відповідь на питання Вови. Фокус:

## Крок 4 — Інструкція Вові

Після виводу код-блоку написати Вові коротко:

> **Готово.** Скопіюй блок вище (кнопка Copy у правому куті) і встав у Gemini (застосунок на телефоні або https://aistudio.google.com з вибраною моделлю **Gemini Pro**). Коли отримаєш відповідь — встав її сюди, я проаналізую і скажу що думаю.

## Важливо

- **НЕ** виконувати реальних змін у коді цим скілом. Скіл тільки збирає контекст і формує промпт (інструкцію).
- **НЕ** включати в промпт `node_modules/`, `bundle.js`, `.git/`, бінарники.
- Якщо git diff виявиться занадто великим — попередити Вову і запропонувати звузити (наприклад тільки останній коміт замість всіх від main).
- Якщо Вова не задоволений відповіддю Gemini — обговорити з ним чому, не просто погоджуватись.
