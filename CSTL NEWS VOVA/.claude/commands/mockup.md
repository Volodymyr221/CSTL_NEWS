# /mockup — ASCII макет UI перед кодом

Аргумент: назва фічі або екрану. Наприклад: `/mockup news-card` або `/mockup buses-tab`

## Що зробити

1. Намалюй ASCII макет (текстове схематичне зображення) нового UI елемента або екрану
2. Покажи різні стани якщо є: порожній стан, заповнений, активний
3. Вкажи розміри якщо критично: ширина, висота, padding (відступи всередині)
4. Поясни логіку: що відбувається при натисканні, свайпі, скролі

## Формат макету

```
┌─────────────────────────┐
│  Назва екрану           │
├─────────────────────────┤
│  [елемент]   [кнопка]  │
│                         │
│  Текст контенту         │
│  підзаголовок           │
└─────────────────────────┘
```

## Після макету

Написати Вовау: "Ось макет. Підтверджуєш — і тільки тоді починаю код."

Не починати писати код поки Вова не сказав "Роби".
