# /start — Швидкий старт нової сесії

**Коли використовувати:** перша команда у кожному новому чаті. Один виклик — і Claude готовий до роботи з повним контекстом проекту.

**Без аргументів.** Просто `/start`.

---

## Крок 1 — Прочитати обов'язкові файли (у цьому порядку)

1. **`START_HERE.md`** (у корені репо) — карта проекту, файлова структура, список скілів
2. **`_ai-tools/SESSION_STATE.md`** — поточний стан: гілка, що робили в останніх сесіях, відкриті задачі
3. **`CSTL_BUGS.md`** (якщо є) — відкриті баги з пріоритетами (🔴 критичні, 🟡 середні, 🟢 дрібні)

**Винятки — НЕ читати:** `CLAUDE.md` (автоматично завантажується як project instructions — інструкції проекту, вже в контексті), `bundle.js` (згенерований файл).

## Крок 2 — Перевірити гілку (branch — гілка розробки)

Виконати:
```bash
git branch --show-current
git log --oneline -5
```

Зрозуміти: на якій гілці працюємо, що в останніх комітах (commit — одна збережена зміна в git).

## Крок 3 — Сформувати коротку відповідь Вові

Написати **4-6 рядків максимум** українською:

1. **Одне речення статусу:** "Прочитав. Проект: CSTL NEWS, гілка [назва], остання робота — [що робили]."
2. **Найкритичніший баг** (якщо є в `CSTL_BUGS.md` з 🔴 пріоритетом) — одне речення
3. **3-5 варіантів що робити далі** — коротко, нумерованим списком. Брати з секції "Відкриті задачі" у `SESSION_STATE.md` або з відкритих багів. Наприклад:
   - `/fix B-01` — виправити конкретний баг
   - Нова функція яку обговорювали
   - Доопрацювання контенту (статті, події)

4. **Запитання:** "Що робимо сьогодні?"

## Крок 4 — Чекати відповідь Вови

**НЕ** починати жодних змін у коді до явного "Роби" від Вови. Це правило №1 проекту (див. `CLAUDE.md` → "Процес роботи").

---

## Приклад результату

> **Прочитав.** CSTL NEWS, гілка `main`, остання сесія — побудували MVP (мінімальний робочий продукт): 4 вкладки, деплой (публікація) на GitHub Pages.
>
> **Критичний баг чекає:** B-01 — деплой падає з помилкою "non-fast-forward" після зміни нікнейму GitHub.
>
> **Варіанти:**
> 1. `/fix B-01` — виправити деплой (рекомендую, терміново)
> 2. Замінити тестові статті на реальний контент
> 3. Переглянути дизайн і покращити
>
> **Що робимо сьогодні?**

## Нагадування для Claude (КРИТИЧНО)

- **Правило №1:** пояснення в дужках до всього не-українською (див. `CLAUDE.md` верхню секцію). Самоперевірка перед кожним повідомленням — обов'язкова.
- **Без "Роби" — код не чіпати.** Тільки обговорення.
- **Короткі відповіді.** Одна думка за раз. Вова на телефоні, довгі повідомлення він не читає.
- **Сперечайся чесно.** Не погоджуйся з Вовою заради ввічливості — вказуй на слабкі місця, пропонуй альтернативи.
