### Фаза 6 — Push-сповіщення «Відстежити маршрут» Level B (Web Push API + Supabase)

**Level A ✅ ЗАВЕРШЕНО (08.06.2026):** in-app банер з снузом, `_notifiedWarning`, `_bannerSnoozedUntil`, `localStorage['bus_track_v1']`.

**Level B — справжні push-сповіщення (навіть якщо додаток закритий):**

**Архітектура (план):**
```
Юзер тапає «Відстежити» на рейсі
    ↓
Браузер запитує дозвіл на сповіщення (Web Push API)
    ↓
VAPID: браузер видає push-subscription endpoint → зберігається у Supabase `push_subscriptions`
    ↓
Supabase pg_cron кожні 5 хв перевіряє рейси
    ↓
За 10 хв до відправлення → Supabase Edge Function → Web Push → телефон
```

**Що потрібно зробити:**
| ☐ | Згенерувати VAPID ключі: `npx web-push generate-vapid-keys` → зберегти у Supabase Secrets |
| ☐ | Таблиця `push_subscriptions` у Supabase: `user_uuid`, `endpoint`, `p256dh`, `auth`, `route_id`, `departure_time`, `date` |
| ☐ | `sw.js` — `push` event listener → `showNotification()` |
| ☐ | `buses.js` — `Notification.requestPermission()` + `serviceWorker.pushManager.subscribe()` → POST до Supabase |
| ☐ | Supabase Edge Function `send-bus-push` — `web-push` npm-бібліотека → надсилає на всі підписки |
| ☐ | Supabase pg_cron — запускає Edge Function кожні 5 хв |
| ☐ | Текст сповіщення: «🚌 ЛУЦЬК → ЛИЧАНИ відправляється через 10 хв (11:30)» |
| ☐ | **Вова має: зайти у Supabase Dashboard → Edge Functions → встановити VAPID_PRIVATE_KEY secret** |

**Примітки:**
- Web Push API безкоштовний, без Firebase/FCM
- Supabase Free tier: 500K Edge Function викликів/міс — вистачить
- iOS підтримує push у PWA тільки з iOS 16.4+
- Сповіщення виглядають як системні — іконка CSTL LIFE + текст
