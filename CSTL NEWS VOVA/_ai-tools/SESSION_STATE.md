## ⚙️ Ключові архітектурні рішення (не змінювати без обговорення)

1. **А+ деплой** — `actions/deploy-pages@v4`, нічого не комітить у `main`
2. **`bundle.js` у git** як робоча база (варіант Б)
3. **`main` як робоча гілка** — пуш = автодеплой
4. **Хук `stop-hook-git-check.sh`** — корисна фіча, не чіпати
5. **Лічильник версії** — `sed` у `deploy.yml`, формат `v{N} · DD.MM HH:MM`
6. **CACHE_NAME у `sw.js`** — оновлювати при КОЖНІЙ зміні. Остання: `cstl-20260416-1311`
7. **Автобуси — зупинки з офіційних квитків VOPAS** — не вигадувати km/ціни без скринів
8. **Світло — дані в `data/power.json`** — зараз DEMO, оновлювати вручну. Схема готова до міграції на Supabase (хмарна база даних)

## ⚡ Модуль «Світло» v2 — поточний стан

### Архітектура
- `data/power.json` — структура: `cities[]{id, name, streets[]}`, `queues[]{id, name, schedule}`. Готова до Supabase
- `src/tabs/power.js` — 3-крокова навігація:
  1. Вибір міста/села (11 сіл, алфавітний порядок)
  2. Вибір вулиці (тільки Олика — 9 вулиць; інші села — автовибір «все село»)
  3. Таймлайн 24 год + .ics-експорт
- `localStorage key`: `power_prefs_v2` (cityId + streetId)
- Кнопка «← назад» для повернення до вибору міста

### Стратегія оновлення даних
| Фаза | Підхід |
|------|--------|
| **Зараз** | Ручне оновлення `data/power.json` через GitHub UI (3 хв) |
| **Середньострок** | Парсер (скрипт що зчитує дані автоматично) — коли Волиньобленерго відкриє стабільні дані |
| **Після Supabase** | Адмін-форма + автопарсер пише напряму в БД (базу даних) |

> ⚠️ Поточні дані в `data/power.json` — DEMO. Замінити реальними даними з Волиньобленерго.
