### Логіка
- **Дедуплікація:** за `sourceUrl` + нормалізований заголовок (між усіма джерелами)
- **Повний текст:** `content:encoded` → якщо < 600 символів → `fetch_full_article()` зі сторінки
- **`fetch_full_article()`** — Chrome User-Agent (BROWSER_UA) + Referer: google.com, CSS-селектори на домен + fallback
- **`strip_html()`** — прибирає «Читати повністю» / «Читати далі» / «Read more» з кінця тексту
- **Класифікатор:** `classify_entry()` → `EVENT_KEYWORDS` + майбутня дата → `events.json` або `articles.json`
- **Cron:** кожні 30 хв (`0,30 * * * *`) — рівно о :00 і :30, надійно виконується GitHub
- **MAX_ARTICLES:** 150, **MAX_PER_SOURCE:** 15, **MAX_EVENTS:** 50

### Виправлення зроблені у цій сесії (2026-04-19)

#### 1. Фікс `NoneType` — корінна причина 20-годинного простою
`feedparser` (бібліотека читання RSS) може повертати `None` всередині списку `entry.content`.
Виправлено в `get_full_content()`:
```python
valid = [c for c in content_list if isinstance(c, dict)]
```
Те саме в `extract_image()` — перевірка `isinstance(m, dict)` перед `.get()`.

#### 2. Захист кожного запису `try/except`
У `parse_source()` — кожен `entry` (запис) обробляється в окремому `try/except`.
Один зламаний запис більше не зупиняє весь парсинг джерела.

#### 3. `BROWSER_UA` — Chrome User-Agent
```python
BROWSER_UA = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/124.0.0.0 Safari/537.36"
```
Використовується в `fetch_full_article()` та `parse_html_source()` + Referer: google.com.
Мета: обійти базові блокування сайтів що не люблять ботів.

#### 4. `strip_html()` — прибирає «Читати повністю»
```python
text = re.sub(r"\s*(Читати повністю|Читати далі|Читати більше|Read more)\s*[»›→]?\s*$", "", text, flags=re.IGNORECASE).strip()
```

#### 5. Cron: `*/10` → `0,30`
`.github/workflows/rss-parser.yml` — `*/10 * * * *` замінено на `0,30 * * * *`.
Причина: GitHub безкоштовний тариф запускає `*/10` лише раз на ~годину.
`0,30` (рівно о :00 і :30) — надійно виконується scheduler (планувальник) GitHub.
