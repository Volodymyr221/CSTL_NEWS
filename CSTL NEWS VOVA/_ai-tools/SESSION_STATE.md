2. **`bundle.js` у git** як робоча база (варіант Б)
3. **`main` як робоча гілка** — пуш = автодеплой
4. **Хук `stop-hook-git-check.sh`** — корисна фіча, не чіпати
5. **Лічильник версії** — `sed` у `deploy.yml`, формат `v{N} · DD.MM HH:MM`
6. **CACHE_NAME у `sw.js`** — оновлювати при КОЖНІЙ зміні статичних файлів. Формат: `cstl-YYYYMMDD-HHMM`. Остання: `cstl-20260416-0002`

## 🚌 Модуль «Автобуси» v3 — поточний стан

### Архітектура (що зроблено)
- `data/schedule.json` — новий формат: `stops[]{name, km, price_from_start}`, `status`, `carrier`, `duration_min`
- **Пошук «Звідки → Куди»** — кастомний dropdown (спадний список) замість ненадійного `<datalist>`
- **Dropdown** — `position: fixed` прив'язаний до пошукової панелі, z-index 500, відкривається під полем
- **iOS-зум виправлено** — `font-size: 16px` на input фільтру всередині dropdown
- **Readonly-поля** — основні поля вибору без клавіатури, тільки tap → dropdown
- **Пропорційний час зупинок** — `departure + round((stop.km / totalKm) * duration_min)`
- **Кнопка «✕ Всі маршрути»** — з'являється тільки коли активний фільтр
- **Акордеон зупинок** — кожна картка рейсу розгортається кнопкою «Всі зупинки»
- **Футер** — двохрядковий, по центру: `VOPAS — Волинська АС-1 · Луцьк / Оновлено: 17:00 | 15 квітня 2026`
