Що змінено: `unsubscribeFromPush` (`buses.js`) тепер симетричний до `subscribeToPush` — повтор 1× + черга «незавершених скасувань» (`localStorage['bus_pending_unsub_v1']`), яку доганяє `flushPendingUnsub()` при відкритті вкладки; `deletePushSubscription` (`supabase.js`) тепер повертає `{ok}`. Edge Function `send-bus-push` НЕ чіпав. Причина: раніше при збої мережі скасування тихо провалювалось → серверний рядок з валідним endpoint лишався → фантомний push на скасований рейс. Перевір з боку Push: (а) чи узгоджується з твоїм баченням каналів; (б) чи не конфліктує черга з майбутнім throttle/муту (П-4).

Репо-версія `supabase/functions/send-bus-push/index.ts` — на гілці `vova/bus-cancel-reliability`, у main потрапить при /finish. **Знай при роботі з Push:** прод-функція вже v11 (не v10 як у main до мерджу).
