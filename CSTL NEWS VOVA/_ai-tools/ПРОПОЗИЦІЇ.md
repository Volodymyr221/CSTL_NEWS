## 🔔 Push _(відп. Рома)_
- [ ] [04.07] Від Вова: **А-3 (надійне скасування автобусних push) зачіпає push-шлях — перепровір при роботі з Push.** Це задача МОЄЇ вкладки «Автобуси» (гілка `vova/bus-cancel-reliability`), але торкається клієнтського push-коду. Що змінено: `unsubscribeFromPush` (`buses.js`) тепер симетричний до `subscribeToPush` — повтор 1× + черга «незавершених скасувань» (`localStorage['bus_pending_unsub_v1']`), яку доганяє `flushPendingUnsub()` при відкритті вкладки; `deletePushSubscription` (`supabase.js`) тепер повертає `{ok}`. Edge Function `send-bus-push` НЕ чіпав. Причина: раніше при збої мережі скасування тихо провалювалось → серверний рядок з валідним endpoint лишався → фантомний push на скасований рейс. Перевір з боку Push: (а) чи узгоджується з твоїм баченням каналів; (б) чи не конфліктує черга з майбутнім throttle/муту (П-4).
