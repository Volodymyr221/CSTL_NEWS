## 📝 2026-07-11 — Погода: актуальна година в скрабері + вирівняний фон (PR #345)

Прямий запит Роми зі скріном (не /byyou): вертикальна лінія в модалці «Погода по годинах» мала показувати актуальну годину за геодаими (фолбек — Київ), і давати перетягувати вліво-вправо щоб бачити іншу годину.

- Скрабер (`community-blocks.js`) вже вмів перетягування — новим був лише автопоказ. Актуальна година рахується з `_wxData.utc_offset_seconds` (Open-Meteo вже віддає її при `timezone=auto`, який враховує реальні координати запиту — геолокацію користувача або фолбек-координати Олики, що дають той самий часовий пояс що й Київ). Автопоказ лише коли відкрито «Сьогодні» — для інших днів немає сенсу «зараз».
- Дрібне UX-покращення в процесі: курсор раніше ховався після відпускання пальця (`clear()` на `pointerup`) — тепер лишається видимим, зручніше звіряти погоду на обрану годину. Мертву `clear()` прибрала.
- Другий запит того ж скріна: фон графіка температури не збігався з графіком опадів (світліший) — вирівняла. Бульбашка опадів дублювала емодзі погоди з бульбашки температури — прибрала (лишила година+%).
- Тест: Playwright з мокнутим `Date`+Open-Meteo API — курсор рівно на очікуваній годині (математично звірено з offset), зникає для не-сьогоднішніх днів, фон обох графіків збігається (`getComputedStyle`), бульбашка опадів без іконки.
- Синхронізувала з Вовиними правками Дошки (модалка НП+дата, фільтр категорій) — нуль перетину рядків, один рутинний конфлікт `sw.js` CACHE_NAME.
