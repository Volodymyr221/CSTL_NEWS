## 🧩 ПОТІК 8 (12.07) — Найближчі події: AI-редактор + Featured від Алли

**Джерело:** розмова з Ромою. Проблема почалась як «фото свят у віджеті Громади — емодзі замість фото», але справжній корінь глибший (з'ясовано дослідницьким агентом, реальний код):

- Пайплайн УЖЕ ІСНУЄ і працює: AI-редактор (`editor/`, місія `holidays`, вікно 7 днів) → чернетка (текст+фото через Wikimedia Commons API — **перевірено цієї сесії, hotlink працює, HTTP 200**, старе застереження в `holidays.json` про блокування Wikipedia стосувалось іншого, гіршого підходу) → кабінет Алли (вкладка `articles` в `admin.html`, НЕ окремий файл) → публікація (`status:'ready'`) → авто-синк (`scripts/sync_cms.py`, крон ~15хв) → `type:news`→`articles.json`, `type:holiday/event`→`events.json` — САМЕ це читає віджет «Найближчі події» (`community-blocks.js:862 renderEventBlock`).
- Тобто свято/подія що пройшли через цей шлях — вже отримують реальне фото автоматично. Проблема лише в охопленні (мало свят у вікні 7 днів, список свят неповний) і в тому що звичайна НОВИНА (`type:news`) ніяк не може потрапити в `events.json` — нема кнопки «показати в Найближчих подіях».

**Уточнення (12.07, після другого раунду питань):** «Найближчі події» — не просто «вітрина з посиланнями», а повноцінна 3-тя категорія в адмінці, поруч з Новини/Свята, з власним потоком чернетка→заплановано→опубліковано. Відбір КАРТОК для віджета — РУЧНИЙ (модератор вирішує), не автоматичний за датою. Хороша знахідка при читанні коду: створення статті типу `event` УЖЕ працює (форма створення вже має перемикач типу Новина/Свято/Подія, `admin.html:1091`) — новий запис із фото не треба «винаходити», просто ЗАРАЗ усі 3 типи звалені в один список без організації. Тому найбільша нова робота — категорійна навігація (перевикористати вже готовий `.tile`-патерн головної сітки, `renderHomeGrid`, `admin.html:785+`) + зручна кнопка «скопіювати опубліковану статтю в Подію» (щоб не передруковувати вручну текст+фото).

**Обсяг розбито на 2 частини:**
- **Частина А** — ширше охоплення AI-редактора (конфіг+дані, малий код)
- **Частина Б** — категорійна вкладка «Статті» (Новини/Свята/Найближчі події) + кнопка «Скопіювати в Найближчі події» (🔴 нова колонка в Supabase `cms_articles`)
