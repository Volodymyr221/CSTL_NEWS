## Корінь (перевірено даними + живими тестами)
14 «обрізаних» новин = усі Олика з Google News. Посилання нового формату (`AU_yq…`)
не розкодовуються простим редиректом → справжній URL видавця невідомий →
`fetch_full_article` не запускається → у стрічку падає лише заголовок. Метод
розкодування (batchexecute) ПЕРЕВІРЕНО наживо: дає справжній rayon-URL, з нього
тягнеться повний текст 718 симв. Головний ризик — блок 429 з IP GitHub Actions →
лікується кешем токенів + тротлінгом.

## ⚠️ ПОВОРОТ АРХІТЕКТУРИ (рішення Роми 05.07) — план вище ЗАСТАРІВ
Замість «ломитись через Google News» → **викидаємо Google News, будуємо AI-агента** (через AI-ключ).
Емпірика: 14 gnews-статей = 9 різних доменів (bbc/ukrinform/zaxid/suspilne…), тож список сайтів
сам покриє лише ~36%. AI web_search дає ті самі широкі знахідки, АЛЕ зі справжніми URL + розумна
фільтрація. Нова система — три «місії» в одному агенті:
- **Громада** (перейменувати з «Олика»): Олика 50% / інші селища громади 50%; теми — люди, пам'ятки,
  населення, культура, події. Потрібна база знань про селища (дослідження запущено).
- **Волинь**: 4-5 джерел; дедуп за ЗМІСТОМ (AI розуміє що новина та сама, навіть іншими словами).
- **Україна та світ**: 5-6 джерел, 50/50; лише серйозне (геополітика/війна/важливе), без «папуг у Таїланді».
Ручний крок Вови/Роми: **Anthropic API-ключ у GitHub Secrets**.
