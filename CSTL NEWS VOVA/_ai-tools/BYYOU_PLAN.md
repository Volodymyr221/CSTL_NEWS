**Ціль:** БЛОК 3 — AI пише лише Громаду+свята → ЧЕРНЕТКИ Аллі (ревʼю). RSS тримає Волинь/Україну/Світ. Палітра тем + памʼять постів/джерел + мʼякий семантичний дедуп.
**Власник:** Рома · **Гілка:** roma/editor-block3 · **Rollback-tag:** byyou-start-block3

## Політика контенту (від Роми)
- **Волинь / Україна / Світ** — НЕ писати AI. RSS-парсер дає (безкоштовно, авто-живе). AI-агент OFF для них.
- **Громада** — AI пише оригінал → **чернетка в кабінет → Алла ревʼю/публікує** (НЕ авто).
- **Свята («Шо в селі»)** — AI пише статтю+фото → **чернетка Аллі** (як свята).
- Обидва AI-написані = через ревʼю Алли (якість/анти-вигадка). Живе лише з ключем Вови.
- Палітра тем + памʼять + дедуп — МʼЯКІ: щоб було різноманіття і без нахабних повторів, але не заважати AI.

## Знахідки коду (реальні)
- `articles.json` 131, ~90% RSS; AI-черга=0. Двигун стрічки зараз = RSS-парсер (cron 30хв): Волинь(volynpost/konkurent)+Україна/Світ(pravda)+Громада-офіц(kivertsi/olytska).
- `ai_news_agent.enqueue()` зараз пише в `news_queue.json`→`publish_queue.py` АВТО-публікує в стрічку. ← міняємо на кабінет-чернетку для Громади.
- `editor/sinks/cabinet.py` уже пише cms_articles draft (реюз для Громади). `sync_cms.py`: type=news→articles.json, holiday/event→events.json (Алла публікує). Потребує ключ Вови.
- `build_prompt` додає лише «ВЖЕ У СТРІЧЦІ» (вікно) — треба персистентну памʼять.

## Принципи

## Примітка про архітектуру (крок 7 = 🔴)
Зміна маршруту Громади з авто-черги на кабінет-чернетку — торкається потоку публікації. Тому крок 7 — 🔴 (стоп/підтвердження перед виконанням). `news_queue.json`/`publish_queue.py` лишаються дормантними (не видаляю, реюз/безпека).
