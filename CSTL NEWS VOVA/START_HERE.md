## 🚀 Як працює деплой (публікація на сервер)

```
Пушиш зміни → GitHub Actions запускається автоматично
  → збирає bundle.js (один файл з усього коду)
  → деплоїть на GitHub Pages (безкоштовний хостинг)
  → vshevchukkk.github.io/CSTL_NEWS (2-3 хвилини)
```

CI (система автоматичного деплою) оновлює час деплою в index.html автоматично.

## 📁 Карта файлів проекту

```
index.html              — Весь HTML (структура сторінки)
style.css               — Всі стилі (зовнішній вигляд)
sw.js                   — Service Worker (офлайн-кешування)
build.js                — Збірка коду (esbuild конфіг)
bundle.js               — Генерується автоматично, не редагувати
data/articles.json      — Статті (новини)
data/events.json        — Події (афіша)
data/schedule.json      — Розклад автобусів
src/app.js              — Точка входу, імпортує всі модулі
src/core/boot.js        — PWA, Service Worker ініціалізація
src/core/utils.js       — Допоміжні функції (час, HTML-захист)
src/tabs/news.js        — Вкладка Новини
src/tabs/events.js      — Вкладка Події
src/tabs/buses.js       — Вкладка Автобуси
src/tabs/submit.js      — Вкладка Подати новину
.github/workflows/      — GitHub Actions (авто-деплой)
```

- Підприємець, не технічний спеціаліст
- Вчиться програмуванню паралельно з роботою над проектом
- Очікує **пояснення кожного технічного терміну** простими словами
- Комунікує коротко, очікує те саме у відповідь
- **"Роби"** = сигнал що можна писати код. Без "Роби" — тільки обговорення
- Скріншоти = основний спосіб показати проблему або ідею
- Допомога потрібна не тільки в IT: мозковий штурм, стратегія, організація проектів
