**Що вже зроблено (Громада, Дизайн v3 D2 «Поле»):**
- Бежева палітра: `--paper #F4F1E6`, `--card-cream #FAF8EF`

## Дизайн-принципи (12 правил для бабусь)

Універсальні правила для **всіх вкладок** (не тільки редизайн):

1. **Один головний об'єкт на екран** — таймер (Світло), наступний автобус, Швидка (Контакти). Не перевантажувати.
2. **Іконка + текст завжди разом** — тільки іконка незрозуміла. Підпис під іконкою або поруч.
3. **Великий шрифт на ключових цифрах** — 28-40px. Допоміжний текст 13-15px.
4. **Високий контраст** — темний текст на світлому. Pastel ОК для фону карток, але текст темно-сірий/чорний.
5. **Червоний — тільки тривога** (немає світла, Швидка, видалення). Все інше — м'які кольори.
6. **Закруглені кути 16-20px** — м'який дружній вигляд.
7. **М'які тіні** `0 4px 12px rgba(0,0,0,0.06)` — глибина без різкості.
8. **Tap-зона мінімум 44×44px** (Apple HIG), для бабусь — 56×56px де можна.
9. **Один main CTA на екран** — головна дія яскравою кнопкою. Решта — текст-посилання.
10. **Мікро-копії з контекстом** — «Дані актуальні на 09:00», «Вхід вільний». Зменшують тривогу.
11. **Loading / empty / error стани** — для кожного екрану. Не порожньо, а м'який текст «Поки нічого. Перевір завтра.»
12. **Pull-to-refresh** на всіх scrolling вкладках — звичка з iPhone.

## Tier 0: Спільні правки для ВСІХ 4 вкладок одразу (1 коміт)

**Це найшвидший win — змінює усі вкладки одним патчем.** Робити ПЕРШИМ.

### Файли які торкатись

| Файл | Зміна |
|------|-------|
| `style/base.css` | Прибрати білий фон body — лишити `var(--bg-page)` (вже з D2) |
| `index.html` | Видалити блоки `<div class="tab-header"><h2 class="tab-title">...</h2><p class="tab-subtitle">...</p></div>` з вкладок `page-news`, `page-events`, `page-buses`, `page-power` (вже зроблено для `page-community`) |
| `style/news.css`, `style/events.css`, `style/buses.css`, `style/power.css` | На всіх `.news-card-*`, `.ev-card`, `.bus-card`, `.pw-*` контейнерах: прибрати `border: 1px solid var(--border)`, додати `box-shadow: var(--card-shadow)`, `border-radius: var(--card-radius)` (16→22px) |

### Очікуваний ефект
- Всі 4 вкладки одразу на бежевому фоні
- 70px вертикалі звільнено (без tab-header)
- М'які тіні замість гострих рамок — узгоджено з Громадою

**Час:** 30-45 хв. Один коміт.

## Tier 1: ⚡ Світло — повний редизайн (НАЙВАЖЛИВІШЕ)

Бабуся щодня дивиться. Велике value, велике спрощення.

### Поточний стан
24 вертикальних рядки з годинами + рожеві/сірі прямокутники «НЕМАЄ»/«є». Excel-таблиця. Око пробігає всі 24 рядки шукаючи зараз.

### Цільовий стан

**1. Hero-таймер зверху (новий блок)**
- Велике коло з прогрес-кільцем (як iPhone Clock Timer)
- Всередині: «До світла: 1 год 7 хв» або «До відключення: 4 год»
- Колір кільця: 🔴 червоний коли немає, 🟢 зелений коли є
- Шрифт 36-40px Georgia bold

**2. Горизонтальна стрічка-термометр на 24 години (замість таблиці)**
- Знизу таймера: 24 кольорових сегментів у ряд
- Зелений = є, рожевий/теракот = немає, сірий = за межами доби
- Маркер-крапка зверху рухається на поточну годину
- Тап на сегмент → tooltip «13:00 — немає світла»

**3. М'яка картка прогнозу на завтра**
- Світло-жовтий фон
- «Завтра: 16 годин без світла. Найдовший період 13:00–21:00»
- Іконка попередження ⚠️

**4. Toggle «Сповіщати про зміни»** (вимагає Фази 6 — Firebase Push)
- Перемикач: коли світло з'являється/зникає — push
- Поки нема push — можна показати disabled з підписом «Скоро»

### Файли
- `src/tabs/power.js` — переписати `renderPowerPage()` (структура, дві нових функції `renderHeroTimer()` + `renderHorizontalTimeline()`)
- `style/power.css` — додати `.pw-hero-timer`, `.pw-timeline-horizontal`, `.pw-tomorrow-card`
- SVG для прогрес-кільця — створити функцію `renderProgressRing(percent, color)`

### Кольори (з D2 палітри)
- `--mint` — є світло (поточне жовте — лишити для індикатора?)
- `--peach` / теракот — немає світла
- `--gray-light` — за межами доби

## Tier 3: 📅 Події — календарна стрічка + структуровані картки

### Поточний стан
Перевантажені картки з фото 16:9, червоним бейджем категорії, текстом. Деякі картки — насправді новини (баг класифікатора RSS, `null` у `time/location`).

### Цільовий стан

**1. Горизонтальна календарна стрічка зверху (нова)**
- 14 наступних днів — маленькі pill з числом і назвою дня
- Крапка на днях де є події
- Тап → фільтрує події на цей день

**2. Структуровані картки подій**

```

│ [17 ТРАВ]   🎉 Свято села           │
│             18:00 · Парк культури   │
│                                     │

│ Вхід вільний.                       │
│                          [Деталі →] │
└────────────────────────────────────┘
```

- Великий день «17» + місяць «ТРАВ» зверху ліворуч (як iOS Calendar)
- Іконка типу події: 🎉 свято / 🕯 пам'ять / 📋 збори / ⚱️ похорон
- Колір фону за типом:
  - Культура/Свято — теракот (`--peach`)
  - Спорт — синій
  - Благодійність — медовий

**3. Чіпи фільтрів** — лишити (Всі/Культура/Спорт/Благодійність), оновити стиль під D2

### Файли

### ⚠️ Баг класифікатора (НЕ дизайн)

## Що НЕ робити

1. **Не торкатися Дошки громади** — її стиль (корковий фон, папірці з цвяшками) **унікальний і теплий**. Він навмисно не «iOS-style». Лишається таким.
2. **Не торкатися Hero-фото Олики** на Громаді — окрема унікальна композиція.
3. **Не змінювати tab-bar внизу** — узгоджений з рештою.
4. **Не змінювати модалку статті** (з sticky-header, swipe) — добре працює.
5. **Не лізти у RSS-парсер** (Python) — окрема зона.

## Питання які треба вирішити перед стартом

1. **Tier 0 робимо одним коммітом чи по вкладці?** Моя рекомендація — одним (швидше + узгодженіше). NM не виключив поодиноко.
2. **Hero-таймер у Світлі** — чи показуємо тільки час до НАСТУПНОЇ зміни (зеленого/червоного), чи додавати інші пов'язані дані (черга, локація — як зараз)?
3. **Push-сповіщення про Світло** — Фаза 6 (Firebase). Toggle ставимо disabled зараз чи приховуємо?
4. **Календарна стрічка у Події** — 14 чи 7 днів? 14 для повноти, 7 для компактності.
5. **Швидка з пульсуючою рамкою** — анімація CSS pulse 2с infinite, чи статична? Бабуся може злякатись постійного «миготіння».
