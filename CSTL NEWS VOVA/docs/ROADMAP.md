### 3.2 — Автоматичне оновлення даних 🔜 НАСТУПНА
**Ситуація:** `energy.volyn.ua` і `voe.com.ua` блокують автоматичні HTTP-запити (403).

**Поточне рішення:** ручне оновлення `data/power.json` через GitHub UI (займає 3 хвилини).

**Стратегія автоматизації (в порядку пріоритету):**
1. **Дослідити** чи публікує Волиньобленерго дані у відкритому форматі (CSV, XML, відкрите API)
2. **Headless-парсер** (Playwright — браузер без вікна) у GitHub Actions cron-job якщо знайдемо стабільну HTML-сторінку
3. **При міграції на Supabase** — парсер пише напряму в таблицю `schedules`, фронтенд читає звідти

> ⚠️ Поточні дані в `data/power.json` — DEMO. Замінити реальними даними з Волиньобленерго.
