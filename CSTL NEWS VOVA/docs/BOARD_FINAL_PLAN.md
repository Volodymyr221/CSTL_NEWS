## Узгоджена концепція (рішення Вови, 13.06.2026)

### Суть
**Громадська дошка** (не OLX-маркетплейс). Папірці на корку (дошка оголошень),
категорії-дії, текст головне, фото опційне, **без ціни/валюти**. Просто, «щоб бабуся змогла».

### Типи постів (було 3 + офіційні → стає 2 + офіційні)
- 🛒 **Оголошення** (`type='board'`) — папірець на корку (стікер).
- 💬 **Обговорення** (`type='chat'`) — картка-прев'ю теми; тап → повноекранна модалка-чат (див. Фаза А2).
- 🏛️ **Офіційні** (`announcements` — окрема таблиця) — від сільради, тільки читання.
- ❌ **ВИДАЛЯЄМО ПОВНІСТЮ:** 🎉 Вітання (`type='greeting'`) і категорію ❤️ Подяка.

### Таби у вкладці Дошка (лишаються три, без змін)
`ДОШКА / ЗБЕРЕЖЕНІ / ОБГОВОРЕННЯ`

### Категорії оголошень (фінальний набір — 6 фільтр-чіпів)
`✦ Всі · 🛒 Куплю/Продам · 🔍 Шукаю · 🔧 Послуги · 🎁 Знайдено/Загубилось · 📢 Оголошення`
- «Куплю» і «Продам» — **один** фільтр-чіп; «Знайдено» і «Загубилось» — **один** чіп.
- **Деталь реалізації (важливо):** пост зберігає **конкретну** категорію
  (`продам`/`куплю`/`знайдено`/`загубилось`) — для правильного лейбла й emoji на стікері.
  А фільтр-чіп **групує** дві конкретні в одну (через масив `match`).

### Контакт покупець↔продавець (двофазно)
- **Зараз (Фаза А):** кнопка «Подзвонити» (`tel:` — відкриває дзвінок) + кнопка Viber
  (deep-link — пряме посилання що відкриває месенджер), якщо автор лишив **номер телефону**.
  Якщо контакт це Telegram-username → кнопка Telegram. Дзвінок лишається назавжди.
- **Фінал (Фаза Б):** справжній внутрішній **приватний** чат у додатку, **тільки для
  авторизованих** (Google-логін). Push-сповіщення через наявну Web Push інфраструктуру.

#### 🔧 УТОЧНЕННЯ ПО КНОПКАХ КОНТАКТУ (рішення Вови 13.06 — обовʼязкові)
1. **Telegram за номером телефону відкрити надійно НЕ можна** (Telegram працює тільки за `@username`).
   Тому:
   - Контакт = **номер телефону** → показуємо тільки **Подзвонити + Viber** (обидва працюють за номером).
   - Контакт = **@username / t.me-посилання** → показуємо тільки кнопку **Telegram**.
   - Інший текст → просто показуємо текст (без кнопок), як зараз.
2. **Вигляд кнопок — ТІЛЬКИ ІКОНКИ** (компактні круглі, як наявна кнопка дзвінка `.cm-board-call`).
   Без підписів словами. 📞 (дзвінок) + 🟣 (Viber) для номера; ✈️ (Telegram) для username.
3. **Deep-link формати:**
   - Дзвінок: `tel:+380...`
   - Viber: `viber://chat?number=+380...` (будується з номера у форматі E.164: `+380XXXXXXXXX`).
   - Telegram: `https://t.me/<username>` (username без `@`).

### Обговорення = тема-чат (зміна 13.06: замість inline-коментарів)
- Картка Обговорення — це **прев'ю теми** (показує останнє повідомлення + лічильник).
- Тап по картці → **повноекранна модалка-чат** (майже на весь екран): стрічка повідомлень
  згори + поле введення внизу.
- **Публічний груповий чат**: усі бачать тему, усі можуть писати й переписуватись.
- Realtime (живе оновлення) — повідомлення зʼявляються одразу в усіх.
- Інлайн-коментарі під карткою — **прибираємо** (їх замінює модалка).

### ⚠️ ДВА РІЗНІ ЧАТИ — не плутати!
- **Обговорення-чат** — ПУБЛІЧНИЙ, усі пишуть, на спільну тему. Робиться у **Фазі А2**.
- **Чат покупець↔продавець** — ПРИВАТНИЙ 1-на-1 на оголошенні, тільки авторизовані. Фаза Б.

### Коментарі під Оголошеннями
- **НЕ потрібні** (тільки реакції + контакт). Лишається як зараз.

### Модель входу (Google-логін, у фіналі — Фаза Б)
- **Читати** — завжди можуть усі (без входу).
- **Будь-яка дія** (подати оголошення, реакція, коментар, чат) — **тільки після входу**.
- ⚠️ **Фазування:** правило гейтингу (gating — обмеження доступу) вмикається ТІЛЬКИ
  коли додано Google-логін. До того — поточна анонімна поведінка (модерація захищає від спаму).

### Модерація (за замовчуванням лишається)
pending → адмін → published (очікує → адмін перевіряє → опубліковано).
Можливо послабити для авторизованих у Фазі Б — відкрите питання.

### Збережені
Зараз — `localStorage` (локальне сховище одного браузера). У Фазі Б з логіном —
синхронізація між пристроями через акаунт (опційне покращення).

## Фаза Б — Кабінет жителя (Google-логін) + приватний чат (на етапі «Громада»)

### 🔑 УЗГОДЖЕНІ РІШЕННЯ ВОВИ (13.06.2026) — основа для реалізації
- **Спроектовано, код НЕ почато.** Будуємо коли робитимемо детально сторінку **«Громада»**.
- **Логін лише для ДІЙ.** Перегляд усього + 📞 дзвінок за номером — вільні, без входу.
- **Тільки Google (Gmail)** через Supabase Auth (`signInWithOAuth({provider:'google'})`).
- **Реєстрація (критерії «пройти»):** Ім'я + Пошта (авто з Google) + **дата народження** (окремий крок).
- **Флоу входу:** при відкритті — екран «Приєднайтесь до громади» → **[Увійти через Gmail]** або
  **[Пропустити]**. «Пропустити» → режим перегляду (read-only), вибір памʼятаємо (localStorage),
  не питаємо щоразу. Будь-яка дія в read-only → мʼяке контекстне «Щоб [дія], увійдіть» (не стіна).
- **Сесія** тримається місяцями (`persistSession:true`).

### Матриця доступу: 👀 гість (read-only) vs ✅ зареєстрований
| Дія | Гість | Зареєстрований |
|---|---|---|
| Переглядати все (Громада/Новини/Події/Автобуси/Дошка), розклад, пошук | ✅ | ✅ |
| 📞 Подзвонити за номером, Поділитися, Зберегти (localStorage) | ✅ | ✅ |
| **Подати оголошення** | ❌ | ✅ |
| **Реакції + коментарі** | ❌ | ✅ |
| **Приватний чат з автором оголошення** | ❌ | ✅ |
| **Відстежити автобус + пуш** | ❌ | ✅ |

### Технічна реалізація (SQL — через Supabase MCP `apply_migration`)
```sql
CREATE TABLE profiles (
  uid UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  name TEXT NOT NULL, email TEXT, birth_date DATE, created_at TIMESTAMPTZ DEFAULT now());
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "own profile read"   ON profiles FOR SELECT USING (uid = auth.uid());
CREATE POLICY "own profile write"  ON profiles FOR INSERT WITH CHECK (uid = auth.uid());
CREATE POLICY "own profile update" ON profiles FOR UPDATE USING (uid = auth.uid());

ALTER TABLE posts ADD COLUMN owner_uid UUID REFERENCES auth.users(id);  -- хто подав (якщо залогінений)
CREATE TABLE threads (
  id BIGSERIAL PRIMARY KEY, post_id BIGINT REFERENCES posts(id) ON DELETE CASCADE,
  author_uid UUID NOT NULL, buyer_uid UUID NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(), last_message_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (post_id, buyer_uid));
CREATE TABLE messages (
  id BIGSERIAL PRIMARY KEY, thread_id BIGINT REFERENCES threads(id) ON DELETE CASCADE,
  sender_uid UUID NOT NULL, text TEXT NOT NULL CHECK (length(trim(text)) BETWEEN 1 AND 2000),
  created_at TIMESTAMPTZ DEFAULT now(), read_at TIMESTAMPTZ);
ALTER TABLE threads ENABLE ROW LEVEL SECURITY;  ALTER TABLE messages ENABLE ROW LEVEL SECURITY;
CREATE POLICY "thread participants read" ON threads FOR SELECT USING (auth.uid() IN (author_uid, buyer_uid));
CREATE POLICY "buyer creates thread" ON threads FOR INSERT
  WITH CHECK (buyer_uid = auth.uid() AND author_uid = (SELECT owner_uid FROM posts WHERE id = post_id));
CREATE POLICY "msg participants read" ON messages FOR SELECT
  USING (EXISTS (SELECT 1 FROM threads t WHERE t.id = thread_id AND auth.uid() IN (t.author_uid, t.buyer_uid)));
CREATE POLICY "msg participants write" ON messages FOR INSERT
  WITH CHECK (sender_uid = auth.uid()
    AND EXISTS (SELECT 1 FROM threads t WHERE t.id = thread_id AND auth.uid() IN (t.author_uid, t.buyer_uid)));

CREATE TABLE user_push_devices (
  id BIGSERIAL PRIMARY KEY, uid UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  endpoint TEXT NOT NULL, p256dh TEXT NOT NULL, auth_key TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(), UNIQUE (uid, endpoint));
ALTER TABLE user_push_devices ENABLE ROW LEVEL SECURITY;
CREATE POLICY "own device manage" ON user_push_devices FOR ALL USING (uid = auth.uid()) WITH CHECK (uid = auth.uid());
CREATE POLICY "service reads devices" ON user_push_devices FOR SELECT USING (auth.role() = 'service_role');
```
**Пуші чату в обидва боки:** тригер `AFTER INSERT ON messages` → через `pg_net` викликає нову Edge
Function `send-chat-push` (за зразком `supabase/functions/send-bus-push/index.ts`) → знаходить
отримувача (учасник треда ≠ sender) → web-push на його `user_push_devices`
`{type:'chat', thread_id, title:<ім'я>, body:<текст>}`. `sw.js` розгалужує по `data.type`, гасить
системне коли вікно видиме (вже є). Клієнт реєструє пристрій логікою з `buses.js` (`subscribeToPush`,
але привʼязка до `uid`, без маршруту).
**Безпека:** анонімний `getAnonId()` непридатний (підробляється) → RLS лише на `auth.uid()`;
тред/повідомлення читають **тільки 2 учасники**; email/телефон у чаті не показуємо (лише імʼя).
**Конфіг (руки Вови):** Google Cloud Console → OAuth client (id+secret); Supabase Dashboard →
Auth → Providers → Google + Redirect URL `https://volodymyr221.github.io/CSTL_NEWS/`.

### Початковий опис (для довідки)

Логін — це **не просто гейт**, а **«Кабінет жителя громади»**: постійна
ідентичність, яку додаток памʼятає. Фундамент під майбутні функції
(подати показники води/світла, звернення до сільради тощо).

### Профіль жителя (реєстрація при першому вході)
- **Імʼя** (100% обовʼязкове) — як звертатись + автозаповнення поля «Імʼя» в оголошеннях.
- **Пошта** (100% обовʼязкове) — база користувачів + вхід через Gmail.
- **Вік** (100% обовʼязкове) — демографія (дорослі/підлітки/...).
  *Зберігати рік народження або вікову категорію, не точне число (не застаріває).*

### Логіка входу
- Кнопка «**Вхід через Gmail**» (Google OAuth — вхід через акаунт Google;
  дає Імʼя + Пошту автоматично).
- Gmail НЕ дає вік → після входу один короткий екран «Доповніть профіль» (вік + підтвердити імʼя).
- Сесія (запамʼятований вхід) тримається у `localStorage` місяцями — не змушуємо входити щоразу.
- Памʼятаємо пристрій: якщо житель свідомо вийшов — лишаємо внутрішній ідентифікатор
  і пропонуємо «Це ви, [Імʼя]? Увійти» (один тап; без авто-входу — безпека на чужому телефоні).
- **Реєстрація blocking чи soft** — відкрите питання (див. нижче).

### Технічна реалізація
1. Google OAuth у фронт + Supabase Auth (вбудована система входу Supabase).
2. Таблиця `profiles` (`auth.uid`, name, email, birth_year/age_group, created_at).
3. Таблиця `messages` (post_id, thread_id, sender=`auth.uid`, text, created_at) + RLS на `auth.uid()`.
4. UI приватного чату: покупець відкриває пост → «Написати» (тільки авторизований) → тред 1-на-1.
5. «Мої оголошення → вхідні» для продавця.
6. Push-сповіщення про нове повідомлення (наявна Web Push інфраструктура — від Автобусів).
7. Гейтинг будь-якої write-дії за логіном; переписати RLS під авторизованих
   (закриває 7 попереджень security-advisor — інструмент Supabase що шукає діри безпеки).

### Майбутнє (поверх кабінету, окремі задачі)
Показники води/світла, звернення до сільради, запис на прийом — усе привʼязане до профілю жителя.

## Логіка взаємодії в чатах «Обговорення» — Фаза Б (спроєктовано 13.06.2026)

> Рішення Вови: повну логіку взаємодії з повідомленнями **відкласти у Фазу Б** (після Google-логіну).
> Причина: зараз чат повністю анонімний — «мої» повідомлення позначаються лише локально
> (`localStorage['cstl-my-comments-v1']`), сервер автора не знає; таблиця `comments` не має
> device-id (на відміну від `reactions`, де є `user_id` через `getAnonId()`). Без довіреної
> ідентичності (`auth.uid()`) видалення/модерація користувачем ненадійні. Зараз видаляти вміє
> тільки адмін (`/admin.html` → вкладка «💬 Коментарі», RLS `is_admin()`).

### Фундамент: ідентичність
- Додати колонку `comments.sender_uid TEXT` (= `auth.uid()` після Google-логіну), nullable для старих записів.
- `addComment(...)` у `src/core/supabase.js` пише `sender_uid`.
- Рендер «мої/чужі» (`getMyCommentIds` у `src/tabs/board.js`) перевести з localStorage на `sender_uid`.

### 4 дії над повідомленням
Усі через одне меню — тап/лонг-прес на бульбашці (`.bd-msg-bubble`, `chatMessagesHtml` у `board.js`)
→ компактний bottom action-sheet. Перевикористати `showToast` + модальні патерни з `docs/DESIGN_SYSTEM.md`.

1. **Видалити власне** — у меню для своїх бульбашок. RLS `comments` DELETE:
   `USING (auth.uid() = sender_uid OR is_admin())` (зараз тільки `is_admin()`).
   Realtime DELETE вже обробляється `subscribeComments` → стрічка перемалюється сама.
2. **Поскаржитись** — у меню для чужих бульбашок. Нова таблиця
   `comment_reports (id, comment_id, reporter_uid, reason, created_at)` + RLS INSERT для авторизованих.
   Нова вкладка у `/admin.html` для перегляду скарг (поряд з наявною «💬 Коментарі»).
3. **Копіювати / Відповісти** — Копіювати: `navigator.clipboard.writeText`.
   Відповісти-цитата: колонка `comments.reply_to BIGINT` → над бульбашкою рендериться цитований фрагмент.
4. **Фільтр матюків** — хелпер `containsProfanity(text)` у `src/core/utils.js` (клієнтський список
   укр/рос образливих слів) → блок при надсиланні з тостом «Будь ласка, без образ».
   ⚠️ Це **єдине, що відсікло б образливий контент ще до публікації** (мотивація зі скріну Вови) —
   варто розглянути як ранній підпункт Фази Б. Опційно дублювати серверним тригером пізніше.

## Чат-модалка — ідеї з промпта ChatGPT (аналіз, 13.06.2026)

> Вова приніс промпт ChatGPT з вимогами до UI чат-модалки. Нижче — розкладка ідей на
> «вже є / швидкі виграші / Фаза Б». Поточний код: `openChatModal`/`chatMessagesHtml`
> у `src/tabs/board.js`, стилі `style/community.css`.

### ✅ Вже є (промпт підтверджує наш напрям)
- Бульбашки Telegram/iMessage, групування підряд одного автора, ім'я+аватар раз на групу.
- Час компактно всередині бульбашки внизу (`.bd-msg-time`) — не окремим рядком.
- Власні = бордові справа, чужі = світлі зліва з аватаром.
- Закриття свайпом вниз; плавна клавіатура (visualViewport + window.resize + debounce).
- Палітра: тепле тло, кремові картки, бордовий акцент, м'які тіні, великі радіуси
  (промпт радить фон `#FAF8EF`, у нас близький `#F3EFE3` — дрібний тюнінг за бажанням).
- Фіксоване поле вводу з кнопкою відправки справа.

### 🟢 Швидкі виграші — можна ЗАРАЗ, без Google-логіну (per-device, як `cstl-my-comments-v1`)
1. **Розумний автоскрол** — `rerenderCommentsBlock` (board.js:949-958) зараз ЗАВЖДИ скролить
   донизу і збиває позицію при читанні старих. Скролити лише якщо користувач біля низу.
2. **Floating-пігулка «N нових повідомлень»** → тап = скрол донизу (коли прийшли нові, а ти вгорі).
3. **Роздільник «↓ Нові повідомлення» + last-seen** — timestamp останнього виходу з теми у
   localStorage (новий ключ `cstl-chat-seen-v1`), при відкритті роздільник перед новими +
   автоскрол до місця прочитаного, далі зникає.
4. **Підрядок у хедері «N відповідей»** — дані вже є (`getComments(post.id).length`).

### 🔵 Потребує більшого — у Фазу Б (або пізніше)
- **«N учасників»** — до логіну ненадійно (всі анонімні «Житель»); точно лише з `sender_uid`.
- **Статус доставки/прочитання** власних — потребує read-receipts, не для анонімів, відкласти.
- **Кнопка «+» (вкладення)** — фото в чаті пізніше (Supabase Storage уже є для Дошки).
