Цей документ є головною інструкцією для розробки. Всі зміни мають відповідати описаним нижче модулям.

## 1. Система ідентифікації та зворотного зв'язку

- **Anonymous ID (анонімний ідентифікатор):** При першому запуску генерувати `deviceId` (UUID — унікальний ідентифікатор пристрою) та зберігати в `localStorage` (локальному сховищі браузера).
- **Внутрішня форма:** Замінити `mailto:` (відкриття поштового клієнта) на `fetch`-запит (HTTP-запит) до API (Web3Forms або EmailJS — сервіси для відправки форм без серверу).
- **Мета:** Відправка новин/відгуків без виходу з PWA. Передавати `deviceId` разом із даними форми.

**Файли:** `src/tabs/submit.js`, `src/core/boot.js`

---

## 2. Модуль "Розумний Транспорт" (Автобуси)

- **Парсинг (автоматичне зчитування) даних:** Реалізувати логіку оновлення розкладу з зовнішніх джерел (сайт автостанції).
- **Tracking System (система відстеження):** Функція "Запланувати поїздку". При натисканні — підписка на конкретний `trip_id` (ідентифікатор рейсу). Зберігати в `localStorage`.
- **Emergency Notifications (термінові сповіщення):** Якщо в отриманих даних рейс позначено як "Скасовано", а користувач його відстежує — негайно вивести Push-сповіщення (через Service Worker — фоновий скрипт браузера).

**Файли:** `src/tabs/buses.js`, `sw.js`, `data/schedule.json`

---

## 3. Модуль "Енерго-Варта" (Світло)

- **Персоналізація:** Вибір району/вулиці або "Черги" користувачем. Збереження вибору в `localStorage`.
- **Dynamic Status (динамічний статус):** Парсинг актуальних графіків (Волиньобленерго — обласна електромережа).
- **UI:** Візуальний статус (Є світло / Немає / Можливе відключення) + таймер зворотного відліку до зміни стану.

**Нові файли:** `src/tabs/energy.js`, `data/energy.json`

---

## 4. Погода та Інтерфейс

- **Smart Weather (розумна погода):** Віджет у хедері (шапці). Пріоритет — геолокація користувача (через `navigator.geolocation` — API браузера для визначення координат). Якщо доступ заблоковано — дефолт на координати Олики.
- **Animations (анімації):** Додати легкі CSS-переходи (0.2s-0.3s opacity/transform) між вкладками та при відкритті модальних вікон статей.
- **History API (API браузерної навігації):** Перехоплення кнопки "Назад" у браузері/смартфоні для закриття модальних вікон без виходу з додатка.

**Файли:** `src/core/utils.js`, `src/app.js`, `style.css`, `index.html`

---

## 5. Технічні вимоги (Stark Standards)

- **Vanilla JS Only (тільки чистий JS):** Жодних важких фреймворків. Максимальна швидкість завантаження.
- **PWA Focus:** Коректна робота в офлайні (кешування останніх успішно завантажених даних).
- **Auto-Sync (автосинхронізація):** Автоматичне оновлення даних (`fetch` — HTTP-запит) при переході пристрою з офлайну в онлайн (подія `online` в браузері).
