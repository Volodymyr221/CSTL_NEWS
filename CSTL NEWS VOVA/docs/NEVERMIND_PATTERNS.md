# NEVERMIND_PATTERNS.md — Рішення взяті з NeverMind

> Цей файл пояснює які конкретні рішення і підходи взяті з проекту NeverMind
> і чому вони корисні для CSTL NEWS.

---

## Що таке NeverMind

NeverMind — персональний PWA-агент продуктивності (задачі, звички, нотатки, фінанси).
Технічний стек: ванільний JS, localStorage, GitHub Pages, без фреймворків.
Розроблявся разом з Claude Code протягом кількох місяців.

CSTL NEWS — інший проект (медіа-платформа), але використовує деякі перевірені технічні рішення з NeverMind.

---

## Взяті рішення

### 1. PWA Setup + Service Worker (офлайн-кешування)

**Звідки:** `src/core/boot.js` в NeverMind
**Де в CSTL NEWS:** `src/core/boot.js`

**Що це:** код який дозволяє сайту:
- Встановлюватись на телефон як повноцінний додаток
- Працювати офлайн (без інтернету)
- Автоматично оновлюватись коли є нова версія

**Чому взяли:** NeverMind вирішив складні проблеми з iOS (iPhone) — наприклад, правильне оновлення кешу при поверненні з фону. Ці фікси (виправлення) перевірені на практиці.

**Ключовий момент:** `CACHE_NAME` (назва кешу) треба міняти при кожному деплої, формат: `cstl-YYYYMMDD-HHMM`

---

### 2. esbuild + модульна архітектура

**Звідки:** `build.js` в NeverMind
**Де в CSTL NEWS:** `build.js` + `src/` структура

**Що це:** esbuild збирає (bundle) всі окремі JS файли в один `bundle.js`. Це дозволяє:
- Писати код в окремих файлах (чистіше, зрозуміліше)
- Браузер завантажує один оптимізований файл

**Конфіг (8 рядків):**
```javascript
require('esbuild').buildSync({
  entryPoints: ['src/app.js'],
  bundle: true,
  outfile: 'bundle.js',
  format: 'iife',
  minify: false,
  sourcemap: true,
});
```

---

### 3. GitHub Actions авто-деплой

**Звідки:** `.github/workflows/auto-merge.yml` в NeverMind
**Де в CSTL NEWS:** `.github/workflows/deploy.yml`

**Що це:** при кожному `git push` (відправці коду) автоматично:
1. Встановлює залежності (`npm install`)
2. Збирає `bundle.js` (`node build.js`)
3. Публікує на GitHub Pages

**Адаптація для CSTL NEWS:** прибрали логіку merge feature-branch (непотрібна), спростили до прямого деплою з main.

---

### 4. Анімація переходів між вкладками

**Звідки:** `animateTabSwitch()` в `src/core/boot.js` NeverMind
**Де в CSTL NEWS:** `window.switchTab()` в `src/app.js`

**Що це:** плавний fade-перехід (0.18s зникає стара вкладка, 0.22s з'являється нова). Виглядає як нативний мобільний додаток, не як звичайний сайт.

---

### 5. Утиліти (допоміжні функції)

**Звідки:** `src/core/utils.js` в NeverMind
**Де в CSTL NEWS:** `src/core/utils.js`

| Функція | Що робить |
|---------|-----------|
| `formatTime(ts)` | Перетворює timestamp в "щойно", "5 хв тому", "2 год тому" |
| `escapeHtml(s)` | Захист від XSS — небезпечні символи в HTML замінює на безпечні |
| `showToast(msg)` | Показує маленьке сповіщення знизу екрану |

---

### 6. settings.json — дозволи і хуки

**Звідки:** `.claude/settings.json` в NeverMind
**Де в CSTL NEWS:** `.claude/settings.json`

**Що це:** налаштування для Claude Code:
- Які bash команди дозволені автоматично
- Які заборонені (наприклад `git push --force`)
- Хуки (автоматичні дії) — наприклад перевірка синтаксису після кожного збереження JS файлу

## Що НЕ взяли і чому

| Що | Чому не взяли |
|----|--------------|
| localStorage для статей | Дані в localStorage видні тільки в одному браузері. Для публічного сайту потрібні JSON файли в репозиторії |
| AI inbox логіка | NeverMind використовує AI для категоризації записів. CSTL NEWS — медіа, не продуктивність |
| Модулі tasks/habits/finance | Не наша функціональність |
| Cross-tab sync | Синхронізація між вкладками браузера — не потрібна для новинного сайту |
| OWL Board (AI агент) | Складна система проактивних повідомлень — можливо в майбутньому |
