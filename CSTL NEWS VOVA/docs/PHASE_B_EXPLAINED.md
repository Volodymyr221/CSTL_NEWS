# Фаза Б — пояснення (навіщо авторизація в Дошці) — для контексту

> Доповнює `BOARD_FINAL_PLAN.md` (там SQL/RLS/план). Тут — «навіщо» простими словами + технічно.

## Чому авторизацію впроваджуємо саме в «Дошка»
Дошка — єдине місце, де люди **щось роблять** (подають оголошення, пишуть, реагують,
а головне — приватно списуються автор↔покупець), а не просто читають. Будь-яка взаємодія
між людьми потребує відповіді «**хто ти?**»: щоб доставити повідомлення саме тій людині,
щоб листування бачили лише двоє, щоб чати не губились при зміні пристрою. Зараз усі анонімні —
тому авторизація це фундамент саме під Дошку.

## Що це дає
1. Приватний чат автор↔покупець (замість «подзвони на номер»).
2. «Мої оголошення» / «Вхідні» для автора.
3. Реальна модерація (за повідомленням стоїть акаунт) → менше спаму/образ.
4. «Видалити своє», лічильник реальних учасників.
5. Збережене синхронізується між пристроями.
6. Фундамент під «Кабінет жителя» (показники води/світла, звернення до сільради).

## Як упроваджуємо (кроки)
- **Крок 0 (руки Вови):** Google Cloud OAuth client + Supabase Google provider.
- **Крок 1 — Вхід:** екран «Приєднайтесь» (Увійти через Gmail / Пропустити) + «дата народження» раз.
- **Крок 2 — Гейтинг:** читати вільно; дія гостя → м'яке «увійдіть». Одна обгортка `requireAuth`.
- **Крок 3 — Приватний чат:** кнопка «Написати автору» → тред 1-на-1 (`threads`/`messages` + RLS).
- **Крок 4 — Push:** нове повідомлення → системне сповіщення (та сама інфра, що автобуси).

## Технічно (стисло)
- **Supabase Auth + Google OAuth** → постійний `auth.uid()` + JWT (підписаний токен, не підробити).
- **RLS (Row-Level Security)** — правила доступу в самій БД на `auth.uid()`; з фронту не обійти.
  Тред/повідомлення читають лише 2 учасники. Саме тому анонімний id не годиться (підробляється).
- **Ідентичність у коді:** `src/core/auth.js` → `currentUserId()` + `requireAuth(label, fn)`.
  Весь код питає лише `currentUserId()` — не залежить від способу входу.
- **Realtime** (WebSocket) для живих повідомлень; **Edge Function `send-chat-push`** +
  тригер `AFTER INSERT ON messages` (через `pg_net`) для пушів; `sw.js` розгалужує push по `data.type`.
- **Стан:** фундамент (`auth.js` + опції клієнта) закодовано на гілці `vova/auth-foundation`
  (не влитий). Лишилось: конфіг, міграції (`profiles/threads/messages/user_push_devices`),
  UI (екран входу, Кабінет, «Написати автору», модалка приватного чату), Edge Function, тригер.
