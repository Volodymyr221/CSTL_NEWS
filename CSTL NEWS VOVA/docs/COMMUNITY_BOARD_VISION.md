# COMMUNITY_BOARD_VISION.md — Дошка громади 2.0

> План розвитку Дошки громади з простих папірців у повноцінну платформу
> «OLX рівня громади Олики».
> Обговорено з Вовою 2026-05-13.

---

## Концепція

**Зараз (v1):** одноформатні папірці на корковій дошці. Текст + ім'я + контакт.
Як школьна дошка оголошень — мило, але обмежено.

**Куди йдемо (v2):** **«OLX рівня громади»** — повноцінна платформа де:
- Жителі купують/продають товари, шукають послуги
- Обговорюють життя села (плітки, питання, новини громади)
- Вітають одне одного (день народження, новонароджені, подяки)
- Знаходять загублені речі/тварин

Стандарти: зручно, легко, зрозуміло, читабельно. Бабуся повинна користуватись.

## Архітектура: 3 типи постів в одному блоці

### Зверху — таби-фільтри

```
🔄 Усі  │  🛒 Дошка  │  💬 Розмови  │  🎉 Вітання
```

Перемикання змінює тип карток у стрічці.

### Структура карток за типом

**🛒 Дошка** — як OLX:
- Фото 1-5 шт (Supabase Storage)
- Заголовок + ціна (грн / «безкоштовно» / «обмін»)
- Категорія: Тварини · Авто · Дім · Послуги · Робота · Знайдено · Загубилось · Інше
- Локація (село ОТГ — Олика / Дерно / Ставок ...)
- Контакт: телефон / Telegram
- ❤️ реакція · 💾 зберегти у «Моє»

**💬 Розмови** — як Telegram-чат, але карточками:
- Аватарка + ім'я (або «анонім» якщо порожнє)
- Текст до 500 символів
- Можна 1 фото
- Теги-хештеги: #громада #дороги #свято #дитсадок
- 👍 / 💬 коментарі (Фаза 2+)

**🎉 Вітання** — святкові картки:
- Великий емодзі/іконка/фото
- Текст
- Кому вітання («Сергію» / «всім матерям»)
- ❤️ реакції

## MVP (Фаза 1 — обовʼязкове)

Має бути у першій версії v2:

| # | Фіча | Чому |
|---|------|------|
| 1 | **Supabase Storage для фото** | Без фото оголошення не працюють |
| 2 | **Категорії** для Дошки + теги-хештеги для Розмов | Інакше каша |
| 3 | **Пошук** зверху | 100+ постів без пошуку не знайдеш |
| 4 | **Реакції** (❤️👍😂😢) — без коментарів спочатку | Швидкий feedback без модерації коментарів |
| 5 | **Збережені** ❤️ → вкладка «Мої збережені» | Повернутись до пост пізніше |
| 6 | **Анонімність на вибір** — порожнє ім'я = анонім | Бабусі простіше |
| 7 | **Модерація обов'язкова** — pending → admin → published | Без сміття і спаму |

## НЕ робити одразу (Фаза 2+)

Це наступні ітерації, не лізти зараз:

- **Чат всередині** (як OLX message-to-seller) — потребує auth + push
- **Профілі користувачів** з історією постів + рейтинг — потребує авторизації
- **Push-сповіщення** про новий пост у категорії — Фаза 6 (Firebase)
- **Геолокація на мапі** — overkill для села
- **Платні підняття** оголошень — потім, коли буде потік
- **Coords-based «поряд зі мною»** — село невелике, всі поруч

## Технічні вимоги

| Що | Як |
|----|-----|
| **БД** | Supabase tables: `posts`, `reactions`, `comments`, `saved` |
| **Storage** | Supabase Storage bucket `community-photos` (1 GB безкоштовно) |
| **Auth** | Для повноти — Telegram login або magic-link email. У MVP — можливо без auth (post → pending → admin) |
| **Модерація** | Адмін-панель `/admin` на сайті: pending list → approve/reject/delete |
| **Realtime** | Supabase Realtime subscriptions → нові пости з'являються без перезавантаження |
| **Search** | PostgreSQL full-text search на title + text |

### Схема таблиці `posts`

```sql
CREATE TABLE posts (
  id          BIGSERIAL PRIMARY KEY,
  type        TEXT NOT NULL,  -- 'board' | 'chat' | 'greeting'
  category    TEXT,           -- для type='board'
  title       TEXT,           -- для type='board'
  text        TEXT NOT NULL,
  price       NUMERIC,        -- nullable
  currency    TEXT DEFAULT 'UAH',
  photos      TEXT[],         -- URL-и зі Storage
  tags        TEXT[],         -- хештеги
  location    TEXT,           -- село ОТГ
  author      TEXT,           -- ім'я або NULL для анонімів
  contact     TEXT,           -- телефон/telegram
  status      TEXT NOT NULL DEFAULT 'pending',  -- pending | published | rejected
  created_at  TIMESTAMPTZ DEFAULT now(),
  published_at TIMESTAMPTZ
);
```

## Ітераційний план

### Спринт 1 — бекенд (1 тиждень)
- Підняти Supabase проект
- Таблиці `posts`, `reactions`, RLS правила
- Перенести поточну дошку (`community-board.json`) у Supabase
- Адмін-панель `/admin` для модерації (pending → published)
- Submit-форма дошки → реально POST у Supabase

### Спринт 2 — фото + типи (2 тижні)
- Storage bucket + upload фото з форми
- Перехід на 3 типи постів (`board` / `chat` / `greeting`)
- Різна структура карток для кожного типу
- Таби-фільтри зверху дошки

### Спринт 3 — пошук + реакції (1-2 тижні)
- Реакції ❤️👍😂😢 + лічильник
- Збережені (per device, без auth)
- Пошук + категорії-чіпи для дошки

### Спринт 4 — авторизація + коментарі (2 тижні, опціонально)
- Magic-link email або Telegram login
- Профілі (аватарка, історія постів)
- Коментарі під розмовами

### Спринт 5+ — масштабування
- Realtime subscriptions
- Push-сповіщення (Фаза 6)
- Платні підняття (Фаза 7)

## Стратегія віральності (як виростити аудиторію до 500+/день)

Обговорено 13.05: у селі сарафанне радіо швидше за рекламу, але є **install
friction** (PWA треба ставити з браузера, бабуся не знає як). Потрібні дві штуки:

### 1. Кнопка 📤 «Поділитися» на кожному типі контенту

Web Share API (`navigator.share()`) — iOS Safari відкриває рідне меню iOS
з Viber/Telegram/Messenger/SMS одним тапом. Без share — copy URL → відкрити
Viber → встромити → надіслати = 5 кроків. З share — 2 тапи.

**Місця кнопки:**
- Модалка статті (новини) — найважливіше
- Папірець на Дошці — конкретне оголошення
- Картка події — «концерт 25 травня → ходімо?»

**Fallback:** якщо share API не підтримується — `navigator.clipboard.writeText`
+ toast «Скопійовано посилання».

Завдання у BACKLOG → 🟡 Незабаром → «Стратегія віральності — Web Share API».

### 2. Перші 10 «адвокатів» поіменно

Не масштаб, а руками. Кожному персонально показати додаток + попросити
у перший день написати один пост:

- Голова громади (постить офіційне → люди йдуть)
- Старости сіл (Дерно, Ставок, Жорнище, Метельне...)
- Активні мами шкільних чатів (батьківські Viber-групи)
- Місцеві блогери (хто веде Олика-Instagram)
- Власники магазинів (Колос → банер на касі)
- Священники (на месі: «у нас є додаток»)

### 3. «Як поставити на телефон» — сторінка `/install`

Окрема сторінка з 3 скріншотами «Як додати на головний екран iPhone».
QR-код у магазинах Олики на касі: «Скануй → постав за 30 секунд».
Заплановано до Дошки 2.0 (Спринт 2-3).
