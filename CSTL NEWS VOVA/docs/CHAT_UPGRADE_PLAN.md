А реалізувати **Local Social Graph** (локальний соціальний граф) — найкращу платформу

- Fast, Predictable, Human

## Етап 1 — Push-система (найактуальніше) 🔴

> Проблема: зараз `send-group-push` шле push на КОЖНЕ повідомлення → спам → люди
> вимкнуть сповіщення. Виправити до того як групи наберуть активних учасників.

### 1.1 Throttle (обмежувач) push для груп
- Максимум **1 push на 5 хвилин** на групу
- Реалізація: у `send-group-push` зберігати `last_push_at` на групу (через таблицю або
  Supabase KV/Edge Config), пропускати якщо менше 5 хв
- Або простіше: DB-поле `chat_groups.last_push_sent_at` + check у Edge Function

### 1.2 Bundling (групування) push
- Замість: "Іван написав" + "Петро написав" + "Марія написала" (3 окремих)
- Стало: **"3 нових повідомлення у «Наша Митильня»"** (1 push)
- Реалізація: рахувати кількість повідомлень з `last_push_at` до поточного → включати
  в текст push

### 1.3 Ієрархія пріоритетів push (архітектурне рішення)

| Рівень | Подія | Правило |
|--------|-------|---------|
| 🔴 Critical | Особисте повідомлення, reply на твоє, запрошення в групу | Миттєво, завжди |
| 🟡 Community | Нове у приватній групі, згадка @, закріплено адміном | 1 push / 5 хв на групу |
| 🟢 Info | Новини, події, зміна розкладу | Не частіше 3 на день |

### 1.4 Налаштування сповіщень (у профілі)
- Окремі тогли (перемикачі) для кожного типу: Особисті / Групи / Новини / Автобуси
- Мут (заглушити) конкретної групи

**Файли:** `supabase/functions/send-group-push/index.ts`, `src/core/supabase.js`,
`src/core/account-ui.js` (налаштування)

## Етап 2 — Груповий чат UX 🟡

> Поточний груповий чат — v1 (тільки текст). Довести до рівня приватного чату.

### 2.1 Закріплені повідомлення (Pinned Messages)
- Адмін може закріпити повідомлення → показується банером зверху в чаті
- БД: `chat_group_messages.pinned boolean` або окреме поле в `chat_groups.pinned_message_id`
- UI: банер під шапкою групи, тап → скрол до повідомлення

### 2.2 Роль Moderator
- Поточні ролі: `admin` / `member`
- Додати: `moderator` — може видаляти чужі повідомлення, але не може змінювати
  учасників і налаштування групи
- БД: розширити `chat_group_members.role` CHECK

### 2.3 Редагування та видалення повідомлень
- **Редагування:** 5 хвилин після відправки (позначка "ред." на повідомленні)
- **Видалення:** автор — без обмежень; модератор/адмін — будь-яке
- RPC `editGroupMessage` / `deleteGroupMessage` вже є в БД — потрібен тільки UI

### 2.4 Reply через swipe (відповідь свайпом)
- Свайп вправо по бульбашці → reply-preview зверху поля вводу
- Вже реалізовано в приватному чаті (`setupBubbleGestures`) — перенести в груповий UI

### 2.5 Реакції в груповому чаті
- Перенести механіку реакцій з приватного чату / обговорень

**Файли:** `src/core/messages-ui.js`, `src/core/supabase.js`, БД міграції

## Етап 5 — Офіційні спільноти (пізніше) 🟢

> Потребує домовленостей з установами. Реалізовувати ТІЛЬКИ коли є реальний
> партнер (Сільрада, Школа тощо) готовий публікувати контент.

### Структура
- 🏛 Олицька селищна рада
- 🏫 Олицький ліцей
- 🏥 Амбулаторія Олика
- 🎭 Будинок культури

### Механіка
- Пише тільки адміністрація установи (broadcast-mode — режим мовлення)
- Читачі: реакції + коментарі (опціонально)
- Схоже на Facebook Pages, НЕ на Telegram-канал

### Алгоритм рекомендацій груп (після 50+ груп)
```
Score = активність × 40% + відповіді × 25% + реакції × 20%
      + нові учасники × 10% + перегляди × 5%
```
Пріоритет: Митильня > Олика > Волинь > Україна

## Що НЕ робимо (свідомий відмова)

| Функція | Причина |
|---------|---------|
| Архів чатів | Зайво для MVP |
| Папки чатів | Зайво |
| Topics всередині груп | Ускладнює UX |
| Голосові повідомлення | Складна реалізація + модерація |
| Боти | Занадто рано |
| Глобальний пошук по всіх чатах | Не потрібен на поточному масштабі |
