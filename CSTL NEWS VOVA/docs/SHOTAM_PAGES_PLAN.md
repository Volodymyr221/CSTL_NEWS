## ✅ ЗАФІКСОВАНІ РІШЕННЯ (22.07.2026)

1. **Назва вкладки — «СТРІЧКА»** (правильно українською; «лента» — калька з рос.). Внутрішній id `shotam` не чіпаємо.
2. **Порядок таб-бару (зліва→направо):** `Стрічка · Обговорення · [Громада] · Дошка · Автобуси`.
   - Логіка Вови: зліва — інформаційна частина, справа — сервіси. Громада — центральна кнопка (замок), лишається по центру.
   - Було: `Автобуси · Дошка · [Громада] · Обговорення · Шо в селі` → новий порядок = дзеркальне перевертання + перейменування.
3. **UI/UX — СУЧАСНО і правильно** (рівень 2020-х, НЕ «Facebook 2006»): чисті картки, повітря, великі фото, кружечки-канали як stories, плавні переходи. Це вимога Вови.
4. **Публікація — як у Facebook, ПРЯМО зі сторінки** (не з кабінету редактора):
   - Власник/адмін сторінки бачить на СВОЇЙ сторінці поле «написати пост» → публікує від імені сторінки, **з підписом автора-людини** («— Ім'я», живо за uid).
   - Кабінет редактора Алли лишаємо ТІЛЬКИ для системної сторінки «Афіша громади» (події/свята).
5. **Ролі/доступ:** сторінки і власників **створює адмін вручну** (v1, щоб не було спам-сторінок). Власник може призначати **адмінів** (теж право писати). Право писати = власник або адмін ЦІЄЇ сторінки. Решта — читати/лайк/коментар.
6. **Дзвіночок = простий перемикач push** (увімк/вимк сповіщення про нові пости сторінки). БЕЗ «вступу в групу».
7. **Лічильник підписників НЕ показуємо** (5 груп, 0–1 підписник — сенсу нема). Підписки зберігаємо лише щоб слати push.

### Ідентичність поста
Пост завжди **від імені сторінки** (аватар+назва зверху) + **підпис автора-людини** внизу («— Ім'я», живо за uid — той самий механізм синхронізації імен, що вже зроблено).

## 🗄️ Модель даних (нові таблиці Supabase)

1. `pages` — сторінка: `id`, `name`, `theme`, `avatar_url`, `banner_url`, `created_at`.
2. `page_admins` — хто веде сторінку: `page_id`, `uid`, `role` ('owner'|'admin').
3. `page_posts` — пост: `id`, `page_id`, `author_uid`, `text`, `image_url`, `created_at`, `deleted_at`.
4. `page_subscriptions` — підписка на push: `uid`, `page_id` (без показу кількості).
5. Лайки/коментарі — **реюз наявного механізму** (reactions/comments), прив'язка до `page_posts`.

## 🧭 ТЕХНІЧНІ ЕТАПИ (розкладка перед «Роби»)

- **Етап 0 — Таб-бар** (лише клієнт): переставити вкладки в новий порядок + перейменувати «Шо в селі»→«Стрічка». Швидко, незалежно, видимий прогрес.
- **Етап 1 — База даних** (SQL, Вова застосовує — MCP відключений): таблиці 1-4 + RLS (row-level security — правила доступу на рівні рядків). Створити вручну 1-2 тестові сторінки + призначити власника.
- **Етап 2 — Стрічка (Екран 1):** горизонтальний ряд кружечків-каналів + вертикальна стрічка постів усіх сторінок (найсвіжіші зверху). Читання/лайк/коментар (реюз). Сучасний UI.
- **Етап 3 — Екран сторінки (Екран 2):** банер+аватар внахлест, назва, стрічка постів лише цієї сторінки, дзвіночок (push-toggle).
- **Етап 4 — Композер поста на сторінці:** для власника/адміна — поле «написати пост» (фото+текст) на своїй сторінці → публікація від імені сторінки з підписом автора.
- **Етап 5 — Push:** сповіщення підписникам про новий пост (реюз інфраструктури чатів; нова Edge Function `send-page-push`).
- **Етап 6 — «Афіша громади»:** події/свята як системна сторінка (міграція наявних `events` у модель сторінок).

**Залежності:** Етап 0 незалежний. 1→(2,3); 4 потребує 1+3; 5 потребує 4. Етап 6 — пізніше.

## 🎯 Суть зміни

**Було (зараз на проді):** «Шо в селі» = афіша — одна централізована стрічка **подій + свят**, наповнюється з одного центру (`data/events.json` + `data/holidays.json`, кабінет редактора Алли + AI-редактор).

**Стане:** вкладка = **стрічка локальних сторінок-каналів** — як лента Facebook, але лише місцеве і структуроване по авторах-сторінках. Кожну сторінку веде свій **куратор** (адміністратор/модератор), той самий, що вже веде відповідну Facebook-групу Олики.

**Ідея-мотивація Вови (дослівно 20.07):** «таким самим чином, як ці всі адміністратори та модератори ведуть групи в Фейсбуці, вони будуть і вести групи в цьому додатку… коли люди будуть переходити в додаток, то й їхня користь тут, щоб вони зразу були, щоб вони тут вели свій блог, свої сторінки». Тобто **пропозиція цінності для кураторів** — перенести свій блог і аудиторію в додаток.

**Модель взаємодії жителя (рішення Вови 20.07):** **«Канали для читання»** — легкий варіант для v1. Житель читає, лайкає, коментує, підписується. НЕ вступає в групу, НЕ пише сам. (Повні спільноти з ролями/модерацією/вступом — це V2, частина вже в коді як приватні групи.)

## 🧩 Три сутності (нові в базі)

1. **Сторінка (канал)** — назва, тематика, аватар (кружечок), банер, куратор(и).
   Приклади реальних кураторів (з Facebook Олики): Відділ культури/молоді/спорту Олицької ОТГ, Олицька міськрада, «Туристична Олика», меценати, активні жителі-блогери.
2. **Пост** — прив'язаний до сторінки (фото + текст + дата). Автор = **сторінка**, не людина.
3. **Куратор** — людина (uid), що має право писати від імені **своєї** сторінки.

Події/свята (те що зараз) **не викидаємо** — стають одним із видів контенту, напр. системна сторінка «Афіша громади».

## 📱 UI — 2 екрани (макет узгоджено з Вовою 20.07)

### Екран 1 — Головна стрічка (тап на вкладку)
- Шапка: назва вкладки + пошук.
- **Горизонтальний ряд кружечків-аватарок груп** (як Instagram stories), з підписами під кружечком. Тап по кружечку → **відкриває окремий екран групи** (НЕ фільтрує стрічку на місці — це рішення Вови, краще за чіпи-фільтри).
- **Вертикальна стрічка постів** усіх сторінок, найсвіжіші зверху. Картка: маленький аватар+назва групи зверху (тап → група) + широке фото + заголовок + час + лайки/коментарі.

### Екран 2 — Сторінка групи (тап по кружечку)
- Кнопка «←» назад + **дзвіночок «🔔 Підписатись»** (підписка = push-сповіщення про нові пости групи).
- Широкий **банер** + великий круглий **аватар** внахлест.
- Назва групи + к-сть підписників.
- **Стрічка постів ЛИШЕ цієї групи**.
- Вихід назад → знову головна стрічка всіх постів.

## 🔧 Що вже є (реюз, не з нуля)

Половина механізму існує:
- **Кабінет редактора** (для Алли) + **AI-редактор** + скрипт `sync_cms`, що публікує контент у стрічку — це фактично «куратор пише → з'являється в стрічці».
- **Запрошення куратора по пошті** (`editor_invites` + тригер на перший вхід через Google) — той самий механізм, що дає доступ Аллі. Новий куратор = один рядок у таблиці.

**Треба додати:**
- Сутність «сторінка» + прив'язку поста до неї (`page_id`).
- У кабінеті: публікація **від імені своєї сторінки**.
- У стрічці: аватар+назва сторінки на кожному пості + екран сторінки.
- Дзвіночок = підписка на push сторінки (реюз push-інфраструктури).
