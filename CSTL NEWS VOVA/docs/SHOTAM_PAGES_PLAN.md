## 🎯 Суть зміни

**Було (зараз на проді):** «Шо в селі» = афіша — одна централізована стрічка **подій + свят**, наповнюється з одного центру (`data/events.json` + `data/holidays.json`, кабінет редактора Алли + AI-редактор).

**Стане:** вкладка = **стрічка локальних сторінок-каналів** — як лента Facebook, але лише місцеве і структуроване по авторах-сторінках. Кожну сторінку веде свій **куратор** (адміністратор/модератор), той самий, що вже веде відповідну Facebook-групу Олики.

**Ідея-мотивація Вови (дослівно 20.07):** «таким самим чином, як ці всі адміністратори та модератори ведуть групи в Фейсбуці, вони будуть і вести групи в цьому додатку… коли люди будуть переходити в додаток, то й їхня користь тут, щоб вони зразу були, щоб вони тут вели свій блог, свої сторінки». Тобто **пропозиція цінності для кураторів** — перенести свій блог і аудиторію в додаток.

**Модель взаємодії жителя (рішення Вови 20.07):** **«Канали для читання»** — легкий варіант для v1. Житель читає, лайкає, коментує, підписується. НЕ вступає в групу, НЕ пише сам. (Повні спільноти з ролями/модерацією/вступом — це V2, частина вже в коді як приватні групи.)

## 🧩 Три сутності (нові в базі)

1. **Сторінка (канал)** — назва, тематика, аватар (кружечок), банер, куратор(и).
   Приклади реальних кураторів (з Facebook Олики): Відділ культури/молоді/спорту Олицької ОТГ, Олицька міськрада, «Туристична Олика», меценати, активні жителі-блогери.
2. **Пост** — прив'язаний до сторінки (фото + текст + дата). Автор = **сторінка**, не людина.
3. **Куратор** — людина (uid), що має право писати від імені **своєї** сторінки.

Події/свята (те що зараз) **не викидаємо** — стають одним із видів контенту, напр. системна сторінка «Афіша громади».

## 📱 UI — 2 екрани (макет узгоджено з Вовою 20.07)

### Екран 1 — Головна стрічка (тап на вкладку)
- Шапка: назва вкладки + пошук.
- **Горизонтальний ряд кружечків-аватарок груп** (як Instagram stories), з підписами під кружечком. Тап по кружечку → **відкриває окремий екран групи** (НЕ фільтрує стрічку на місці — це рішення Вови, краще за чіпи-фільтри).
- **Вертикальна стрічка постів** усіх сторінок, найсвіжіші зверху. Картка: маленький аватар+назва групи зверху (тап → група) + широке фото + заголовок + час + лайки/коментарі.

### Екран 2 — Сторінка групи (тап по кружечку)
- Кнопка «←» назад + **дзвіночок «🔔 Підписатись»** (підписка = push-сповіщення про нові пости групи).
- Широкий **банер** + великий круглий **аватар** внахлест.
- Назва групи + к-сть підписників.
- **Стрічка постів ЛИШЕ цієї групи**.
- Вихід назад → знову головна стрічка всіх постів.

## 🔧 Що вже є (реюз, не з нуля)

Половина механізму існує:
- **Кабінет редактора** (для Алли) + **AI-редактор** + скрипт `sync_cms`, що публікує контент у стрічку — це фактично «куратор пише → з'являється в стрічці».
- **Запрошення куратора по пошті** (`editor_invites` + тригер на перший вхід через Google) — той самий механізм, що дає доступ Аллі. Новий куратор = один рядок у таблиці.

**Треба додати:**
- Сутність «сторінка» + прив'язку поста до неї (`page_id`).
- У кабінеті: публікація **від імені своєї сторінки**.
- У стрічці: аватар+назва сторінки на кожному пості + екран сторінки.
- Дзвіночок = підписка на push сторінки (реюз push-інфраструктури).
