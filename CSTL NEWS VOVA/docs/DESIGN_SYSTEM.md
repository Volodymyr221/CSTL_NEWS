## ⚠️ Помилки які були зроблені — читати перед будь-якою зміною стилів

### Проблема: padding на дочірніх елементах не звужував вміст

**Що сталось:** Вова просив додати горизонтальні відступи (відстань від краю) всередині модалок. Було зроблено кілька спроб — жодна не давала видимого результату.

**Що реально вирішує проблему:**
```css
/* НА OUTER PANEL (той що має overflow:hidden і border-radius): */
padding: 0 20px;

/* scroll container (контейнер прокрутки) — тільки вертикальний padding: */
padding: 28px 0;
```

**Правило:** padding (внутрішній відступ) завжди ставити на **outer panel** (зовнішня панель — та що має `overflow:hidden`), а не на scroll-контейнері всередині.

## Модальне вікно (Bottom Sheet Modal — панель що виїжджає знизу)

> Заповнити після першої роботи з модалками

### Поведінка
- Закривається тапом (натисканням) по backdrop (затемнений фон за модалкою) або свайпом (рухом пальця) вниз
- `max-height: 85vh` + scroll — якщо вміст не влізає на екран
- `env(safe-area-inset-bottom)` в нижньому padding — для iPhone з home indicator (риска внизу екрану)

### CSS-специфікація (специфікація — точний опис стилів)

```css
/* Зовнішній контейнер (overlay — затемнення) */
position: fixed; inset: 0; z-index: 200;
background: rgba(0,0,0,0.5);

/* Панель модалки */
position: fixed; bottom: 0; left: 0; right: 0;
border-radius: 20px 20px 0 0;
overflow: hidden;
max-height: 85vh;

/* Вміст (content) */
overflow-y: auto;
padding: 28px 0 calc(env(safe-area-inset-bottom) + 28px);
```
