# CONTENT_STRATEGY.md — Стратегія контенту CSTL NEWS

> **Ціль:** зробити стрічку, яка **рятує** жителя від хаосу в Viber, а не додає до нього ще один шум.
> **Принцип:** краще 10 перевірених новин за день, ніж 100 сирих.

---

## Стратегія одним абзацем

Наповнення — **гібридне**: автоматичний парсинг RSS з перевірених волинських та національних джерел + ручна публікація ексклюзивів Олики від редакції. Автоматика дає обсяг і регулярність. Редакція дає унікальність і довіру. Разом — інформаційна монополія в громаді.

---

## Структура стрічки на головній

```
┌────────────────────────────────────┐
│ 🔥 ГАРЯЧЕ                          │  ← ручна закріпка від власника
│ [1 картка, найважливіша зараз]     │
├────────────────────────────────────┤
│ 🏛 ОЛИКА                            │  ← ексклюзиви редакції
│ [2-3 свіжі картки]                 │
├────────────────────────────────────┤
│ 🌾 ВОЛИНЬ                           │  ← RSS, відфільтроване
│ [до 10 карток за день]             │
├────────────────────────────────────┤
│ 🇺🇦 УКРАЇНА                         │  ← топ-5 на день, не більше
│ [3-5 найважливіших карток]         │
└────────────────────────────────────┘
```

**Обов'язкові обмеження:**
- **НЕ заливати усе автоматично.** Чим більше сміття — тим менше довіри.
- **Максимум 20 нових карток за день** (разом усі категорії). Якщо більше — значить фільтр треба посилити.
- **Україна ≤ 5 карток за день.** Повна, але не шумна картина.
- **Гаряче = максимум 1 картка.** Ручна. Власник натискає "закріпити".

## Дедуплікація

Якщо 3 джерела пишуть про одне (наприклад, "Вночі була тривога"):
1. Порівнюємо перші 50 символів заголовків → якщо схожість >70% → вважаємо однаковим.
2. З двох однакових беремо:
   - Пріоритет джерела вищий (1 > 2 > 3)
   - Якщо рівний — свіжіший `ts`
   - Якщо рівний — довша стаття (більше `content.length`)

## Ручна публікація — як власник додає новину

### Варіант на зараз (щойно сайт оживе) — пряме редагування `data/curated.json`

1. Зайти на `github.com/Volodymyr221/CSTL_NEWS/blob/main/data/curated.json`
2. Натиснути олівець "Edit" справа вгорі
3. Додати об'єкт у масив:
   ```json
   {
     "id": 100,
     "title": "Заголовок",
     "excerpt": "Короткий опис для картки",
     "content": "Повний текст статті",
     "category": "Культура",
     "geo": "Олика",
     "image": null,
     "source": "CSTL NEWS",
     "sourceUrl": null,
     "exclusive": true,
     "ts": 1743800000000
   }
   ```
4. Коміт з повідомленням "news: заголовок статті"
5. Автодеплой через 2-3 хвилини → стаття на сайті

**Важливо:**
- `id` має бути унікальним (наступний номер)
- `ts` — це timestamp в мілісекундах. Для "зараз" можна взяти з https://www.unixtimestamp.com/
- `geo` — тільки одне з: "Олика", "Волинь", "Україна", "Світ"
- `exclusive: true` показує червону стрічку "Ексклюзив"

### Варіант майбутнього (Фаза 7) — Telegram-бот

Пишеш боту: `/publish Заголовок | Короткий опис | Повний текст | Олика`
Бот додає у JSON через GitHub API, коміт, деплой. 30 секунд з телефону.

### Інструкція для нових членів команди

Короткий HOWTO — у `docs/HOW_TO_PUBLISH.md` (буде створено у Фазі 2.1).

## Технічна реалізація (Фаза 2.1)

**Файли:**
- `scripts/fetch_news.py` — Python-скрипт з парсингом RSS, фільтрацією, дедуплікацією. Залежності: `feedparser`, `beautifulsoup4` (для очистки HTML).
- `.github/workflows/fetch_news.yml` — cron-job запуск щогодини, виконує скрипт, комітить оновлений `data/articles.json`.
- `data/curated.json` — ручні ексклюзиви, **не чіпається** автоматикою. При зборі фінального `articles.json` — мерджиться.
- `data/articles.json` — фінальна стрічка. Генерується автоматично, **не редагується руками**.

**Принцип мерджу:**
```
articles.json = curated.json + фільтрований RSS-дамп
                ↓
                сортування за ts
                ↓
                обмеження за лімітами категорій
```

**Частота оновлення:** щогодини. Якщо власник додає в `curated.json` ручну статтю — вона одразу потрапляє на сайт при наступному запуску cron (максимум через годину), без чекання парсингу RSS.

## Що НЕ робимо (важливі "ні")

- **Не заливаємо** усе що знайдемо — краще менше, але перевіреного
- **Не копіюємо** повні тексти без атрибуції (ставимо `sourceUrl` завжди, і в модалці статті — кнопку "Читати оригінал")
- **Не публікуємо** неперевірені чутки у категорії "Олика" — тільки те, що підтверджено
- **Не показуємо** рекламу у стрічці новин без мітки "Реклама"
- **Не використовуємо** кричущі заголовки типу "ШОК" навіть якщо їх мають джерела — переписуємо
- **Не ставимо** `exclusive: true` на статті з RSS — тільки на ручні ексклюзиви редакції
