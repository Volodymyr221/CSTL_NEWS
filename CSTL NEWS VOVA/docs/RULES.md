## Пояснення термінів (ОБОВ'ЯЗКОВО)

Після КОЖНОГО англійського слова, терміну, команди — одразу в дужках пояснення українською.

```
✓ ПРАВИЛЬНО:
"Оновив CACHE_NAME (назву кешу — версію збережених файлів) в sw.js (Service Worker — файл що керує кешем)"

✗ НЕПРАВИЛЬНО:
"Оновив CACHE_NAME в sw.js"
```

Вова вчиться паралельно з роботою. Пояснення важливіші за стислість.

## Якість виконання

- **CSS/layout — описати результат перед пушем:** "елемент Xpx, padding Y → вміст буде X−2Y px"
- **Якщо перша спроба не спрацювала — СТОП.** Перечитати код, зрозуміти ЧОМУ, тільки тоді наступна спроба
- **Не губи попередні фікси** — при кожній зміні перевіряти що попередні виправлення збережені
- **Перевіряй синтаксис** — після JS змін: `node --check файл.js`

## Документація

- **Після КОЖНОЇ зміни** — оновити `_ai-tools/SESSION_STATE.md`
- **Після структурних змін** — оновити `CLAUDE.md`
- **Після змін архітектури** — оновити `docs/ARCHITECTURE.md`
- **При додаванні нових файлів** — додавати в `src/` та імпортувати в `src/app.js`

---

## Деплой (публікація)

- **При кожному деплої** — змінити `CACHE_NAME` в `sw.js` (формат: `cstl-YYYYMMDD-HHMM`)
- Використовуй `date` в терміналі щоб дізнатись поточний час
- **bundle.js не комітити вручну** — він генерується CI автоматично
- **Якщо деплой не спрацював:**
  ```
  git commit --allow-empty -m "ci: retrigger" && git push origin main
  ```

---

## Що не можна змінювати без обговорення

- Структуру JSON файлів в `data/` без міграції існуючих даних
- Порядок імпортів в `src/app.js`
- Логіку `setupSW()` в `src/core/boot.js` — кожна частина вирішує конкретну проблему
- `localStorage` для зберігання статей — тільки `data/*.json` файли

---

## Чеклист аудиту (коли Вова каже "перевір роботу")

1. Перечитати всі змінені ділянки коду
2. `node --check файл.js` для кожного зміненого JS файлу
3. Перевірити баланс `<div>` тегів в HTML
4. Шукати дублікати функцій
5. Шукати залишки старого коду і мертві змінні
6. Перевірити що всі нові обробники зареєстровані
7. Порахувати рядки — попередити якщо файл >500 рядків
