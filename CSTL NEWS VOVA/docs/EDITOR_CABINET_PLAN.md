## Рішення (Рома, 01.07)

Будуємо **Кабінет редактора** (варіант B) — веб-форма публікації новин / статей / подій / історій для Алли.

**Контекст рішення:**
- Алла — контент-менеджерка проекту (відео, статті, медіа-робота). Події громади створюватиме переважно вона (див. `CONTENT_STRATEGY.md` → «Команда контенту»).
- Наявна адмінка `admin.html` — **тільки для Дошки громади** (модерація постів мешканців, оголошення, коментарі, адміни). Публікації новин/статей там НЕМА.
- Новини/події/історії зараз живуть у git-файлах (`data/articles.json`, `data/events.json`, `data/olyka-stories.json`) — редагувати їх з веб-форми без бекенду неможливо. Єдиний обхід — правка JSON через GitHub UI (технічно, не для Алли).
- Тому найчистіший шлях: **перенести редакційний контент у Supabase** (як Дошку) і дати Аллі форму. Деталі залежать від відповідей нижче.

**Відхилений варіант A (тимчасовий):** інструкція + правка JSON через GitHub — лишається запасним, поки кабінет не готовий.

## 🔧 Технічні питання (рішення про будову кабінету — вирішує Вова/Рома)

1. Де зберігати новини/події — лишити файли git (JSON) чи перенести в Supabase (база, як Дошка)? Форма з кабінету може писати тільки в Supabase.
2. Вхід у кабінет — через Google (як користувачі) чи окремий логін email+пароль (як admin.html)?
3. Кабінет — розширити наявний admin.html (додати таби Новини/Події/Історії) чи окрема сторінка editor.html?
4. Модерація — Алла публікує одразу чи її пости спершу схвалює хтось (Рома/Вова)?
5. Відео — дозволяємо лише посилання (YouTube/Instagram) чи закладаємо завантаження відеофайлів (дорожче, ліміти Supabase Storage)?
6. Чернетки + планування публікації на дату — закладаємо зараз чи потім?

## 📝 Питання до Алли (що і як публікуватиме — з'ясувати на зустрічі наступного тижня)

1. Які типи контенту робитимеш: новини? довгі статті? події громади? історії Олики? фоторепортажі? відео-огляди?
2. Чи будуть відео — і звідки: посилання на YouTube/Instagram чи власні відеофайли?
3. Поля новини: заголовок, короткий опис, повний текст, фото, категорія, дата — усе? Ще щось?
4. Поля події: назва, дата, час, місце, опис, афіша-фото — цього достатньо?
5. Фото: додаватимеш з телефону (камера/галерея)? Скільки на один пост — одне чи кілька?
6. Працюватимеш з телефону чи з компʼютера?
7. Підпис автора — твоїм ім'ям (автор: Алла) чи від «CSTL LIFE»?
8. Які рубрики/теми потрібні (Культура, Спорт, Освіта, Люди Олики…)?

**Головні три, без яких не почати:** (1) які типи контенту, (2) чи потрібне відео і як, (3) публікує сама чи через модерацію.
