> **Наша реальність (фільтр для всього нижче):**
> - Гіперлокальна громада містечка **Олика** (не глобальний SaaS).

> - Рів проекту: жива локальна громада + гіперлокальні дані (автобуси, світло, події) + мережа.

## 🟢 ВИСОКИЙ ПРІОРИТЕТ — прямо під нашу модель

### SEO (головний канал безкоштовного трафіку для гіперлокалі)
- **`programmatic-seo`** (програмне SEO — масова авто-генерація сторінок) — **топ-важіль.**
  Окрема сторінка під кожну подію / оголошення / автобусний маршрут / «що відбувається в Олиці сьогодні».
  Гіперлокальні запити мають низьку конкуренцію → ми можемо домінувати в Google по «Олика …».
- **`schema`** (структуровані дані — розмітка JSON-LD) — розмітити події як `Event`,
  бізнеси як `LocalBusiness`, новини як `NewsArticle`/`Organization`. → потрапляння в
  розширені результати Google + Google Maps. Прямо підсилює продаж квитків.
- **`site-architecture`** (архітектура сайту під SEO) — логічна структура URL/розділів,
  внутрішні посилання, щоб Google добре індексував.
- **`seo-audit`**, **`ai-seo`** — ✅ уже маємо як скіли. Періодично проганяти.

### Зростання громади (word-of-mouth у містечку = найсильніше)
- **`referrals`** (реферальна програма — «приведи сусіда») — у малому місті сарафанне радіо
  працює найкраще. Механіка: запросив N сусідів → бонус (рання афіша, бейдж, знижка на квиток).
- **`community-marketing`** (маркетинг через спільноту) — наша суть. Залучати лідерів думок
  Олики, місцеві групи Facebook/Viber, школу, церкву, раду.
- **`free-tools`** (безкоштовні інструменти як гачок) — трекер автобусів і графік світла
  ВЖЕ є наші гачки. Скіл підказує як їх використати для залучення + які ще зробити.
- **`lead-magnets`** (магніти — що дати в обмін на підписку/контакт) — напр. push-дайджест
  «головне в Олиці за тиждень», сповіщення про відключення світла.

### Утримання й активація
- **`onboarding`** (знайомство нового користувача) — перші 30 секунд: новий житель одразу
  бачить цінність (новини + афіша + автобуси + світло). Зараз цього немає — кандидат №1.
- **`churn-prevention`** (утримання — щоб не йшли) — повертати через push/корисні нагадування.
- **`marketing-psychology`** (психологія — тригери дії) — соц.доказ («N жителів читають»),
  дефіцит («лишилось X квитків»), локальна приналежність.

### Монетизація (Фаза 5)
- **`offers`** (акції/спецпропозиції) — для локальної комерції на платформі.
- **`pricing`** (ціноутворення) — ціни на квитки й розміщення оголошень/акцій бізнесів.
- **`co-marketing`** (спільний маркетинг) — **крос-промо всередині екосистеми Olyka Castle**
  (Kino_Castle, Smoke_Castle) + партнерства з місцевими бізнесами. Дешево й логічно.

### Аналітика й експерименти
- **`analytics`** — ✅ маємо. **`ab-testing`** (A/B-тести) — перевіряти зміни на даних.
- **`customer-research`** (дослідження аудиторії) — прості опитування жителів: що бракує.

## 🟡 СЕРЕДНІЙ ПРІОРИТЕТ — корисне, але пізніше / з адаптацією

- **`content-strategy`**, **`copywriting`**, **`copy-editing`**, **`social`**, **`launch`**,
  **`marketing-ideas`** — ✅ уже маємо як скіли. Тримати актуальними під укр. контент.
- **`emails`** / **`sms`** — розсилки. У нас push дешевший; email/SMS — пізніше (SMS дорогий).
- **`popups`**, **`signup`**, **`cro`** (Conversion Rate Optimization — оптимізація конверсії) —
  для підписки/реєстрації. Акуратно: не дратувати спливними вікнами.
- **`image`**, **`video`** — візуал для соцмереж і афіші.
- **`directory-submissions`** (реєстрація в каталогах) — локальні каталоги Волині, Google Business.
- **`public-relations`** (зв'язки з громадськістю) — місцева влада, школи, районні ЗМІ.
- **`competitors`** / **`competitor-profiling`** — хто ще «покриває» Олику (Facebook-групи, газети).
- **`marketing-plan`**, **`product-marketing`** — стратегічні, для зведення всього докупи.

## 🔴 НИЗЬКИЙ ПРІОРИТЕТ / неактуальне — наша модель інша

- **`ads`**, **`ad-creative`** — платна реклама. Наша модель **не реклама** → не пріоритет.
- **`cold-email`**, **`prospecting`**, **`revops`**, **`sales-enablement`** — B2B-продажі (SaaS). Не наш кейс.
- **`paywalls`** (платний доступ за стіною) — поки контент безкоштовний (доступ = громада).
- **`aso`** (App Store Optimization) — ми PWA, не в магазинах застосунків (поки).

## Як це використовувати далі
1. Коли беремось за маркетингову задачу — звіряємось із цим списком: що вже є, що взяти з репо.
2. Будь-який скіл звідти **адаптуємо** під укр. мову + локальний контекст + модель «громада + квитки».
   Не копіювати дослівно (там англомовний SaaS-маркетинг).
3. Найперші кандидати на впровадження (за впливом/вартістю): **programmatic-seo + schema**
   (трафік), **onboarding** (активація), **referrals + community-marketing** (зростання).
