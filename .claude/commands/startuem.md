# /startuem — Швидкий старт нової сесії CSTL NEWS

**Коли використовувати:** перша команда у кожному новому чаті проекту CSTL NEWS. Один виклик — і Claude готовий до роботи з повним контекстом проекту.

**Без аргументів.** Просто `/startuem`.

**Відрізняється від `/start`:** скіл `/start` — інший, глобальний, для інших проектів Вови. Не чіпати. Для CSTL NEWS використовується саме `/startuem`.

---

## Крок 0 — Хто на зв'язку (Рома чи Вова) + гілка + конфлікти

> Над проектом працюють ДВОЄ власників — **Вова і Рома**, кожен зі свого Claude. Зроби це ПЕРШИМ, автоматично, нікого не питай. Повна процедура — у `NEW_SESSION_PROMPT.md` (Крок 0). Коротко:

1. **Визнач людину** за поштою сесії (контекст, блок `userEmail`): `roma.haranin.ru1@gmail.com` → **Рома**; інша → **Вова**.
2. **Підпиши коміти:** `git config user.name "<Рома/Вова>"` + `git config user.email "<пошта>"`.
3. **Гілка під людину:** `roma/<тема>` або `vova/<тема>` (потік: іменована гілка → `/finish` → `main`). Авто-гілку `claude/startup-...` лишай до першого «Роби».
4. **Перевір конфлікти:** `git fetch origin` + `git log --oneline --all -15` + кінець `SESSION_STATE.md`/`BACKLOG.md`. Якщо рішення іншої людини суперечать — скажи про це у першій відповіді.
5. **Взаємне автостеження (для ОБОХ власників):** запусти Monitor `SELF='<твій regex>' bash .claude/owner-watch.sh` (persistent) — повідомляє про активність ІНШОГО власника. SELF: Рома=`Рома|roma\.haranin\.ru1@gmail\.com`, Вова=`Вова|Vova|VShevchukkk|Volodymyr221`. Деталі — `NEW_SESSION_PROMPT.md` Крок 0 п.5.

> ⚠️ **Знай про auto-merge:** пуш у будь-яку гілку `claude/**` автоматично мерджиться в `main` + деплой (`.github/workflows/auto-merge.yml`, `-X theirs`). Хочеш контрольований потік через `/finish` — працюй на `roma/...`/`vova/...` (вони НЕ автомерджаться).

## Крок 1 — Прочитай стартовий промпт

Відкрий через Read tool файл:
```
CSTL NEWS VOVA/_ai-tools/NEW_SESSION_PROMPT.md
```

Усередині цього файлу — повний стартовий промпт (сам текст між маркерами ⬇️/⬆️). Він містить:
- Список файлів які ти маєш прочитати
- Ключові правила роботи з Вовою
- Стиль комунікації (пояснення термінів у дужках, короткі повідомлення, сперечатись чесно)
- Формат очікуваної першої відповіді

**Виконуй інструкції з цього файлу як свої.** Він для тебе — офіційний контракт на всю сесію.

## Крок 2 — Виконай обов'язкове читання проекту

Згідно з інструкцією у `NEW_SESSION_PROMPT.md`, прочитай у такому порядку:

1. `CSTL NEWS VOVA/HOT_RULES.md` — 8 болючих правил що порушуються найчастіше (читай ПЕРШИМ)
2. `CSTL NEWS VOVA/START_HERE.md`
3. `CSTL NEWS VOVA/_ai-tools/SESSION_STATE.md`
4. `CSTL NEWS VOVA/_ai-tools/BACKLOG.md`
5. `CSTL NEWS VOVA/CSTL_BUGS.md`
6. `CSTL NEWS VOVA/CLAUDE.md`
7. `CSTL NEWS VOVA/ВОВА_ПРОФІЛЬ.md`

Додатково — подивись git-історію:
```
git log --oneline -10
git branch --show-current
```

Щоб зрозуміти що відбувалось нещодавно і на якій гілці зараз працюємо.

## Крок 4 — Чекай "Роби" від людини (Рома або Вова)

**НЕ** починай жодних змін у коді поки людина не скаже явне **"Роби"**. Це правило №1 проекту (див. `CSTL NEWS VOVA/CLAUDE.md` секція "Процес роботи").

Якщо не впевнений чи фраза — це "Роби" чи просто обговорення, перепитай: **"Це Роби на [назва задачі]?"**. Краще уточнити ніж зробити зайве.

## Нагадування перед стартом

- **Пояснення англомовних термінів у дужках** — у кожному повідомленні, без винятків. Вова вчиться програмуванню через цей проект.
- **Короткі повідомлення** — одна думка за раз.
- **Сперечайся чесно** — не погоджуйся заради ввічливості.
- **Автокоміт через stop-hook — потрібна фіча**, не переписуй.
- **Двогілковий потік:** робота на іменованій гілці (`roma/...` або `vova/...`) → `/finish` (PR → squash) → `main`. У `main` напряму не пушити.

Повний список правил — у `CSTL NEWS VOVA/CLAUDE.md` секція "ВСІ ПРАВИЛА".
