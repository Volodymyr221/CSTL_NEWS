# /start — Швидкий старт сесії CSTL NEWS (репо-локальний алиас `/startuem`)

**Це локальний `/start` для репозиторію CSTL NEWS.** Виконує ТУ САМУ логіку що `/startuem` — окремий скіл НЕ дублюємо, джерело правди одне: `NEW_SESSION_PROMPT.md`.

> ⚠️ **Поза CSTL** глобальний `/start` (для інших проектів Вови) — це НЕ цей файл і він не зачеплений. Тут, у CSTL, `/start` == `/startuem`.

## Що робити
1. Прочитай через Read tool: `CSTL NEWS VOVA/_ai-tools/NEW_SESSION_PROMPT.md`.
2. Виконай усе що там як свої інструкції:
   - **Крок 0:** власник за поштою сесії (`roma.haranin.ru1@gmail.com`→Рома, `vitocorleone191@outlook.com`→Вова [другий акаунт Вови], інша→Вова) → підпис комітів цією людиною → безпечна гілка власника **АБО** продовження `paused` /byyou з тієї ж гілки → `git pull origin main` → перевірка конфліктів → автостеження (Monitor) → заяви активність у `BOARD.md`.
   - Обов'язкове читання проекту (HOT_RULES → START_HERE → BOARD → свій SESSION_STATE → AUDIT → CSTL_BUGS → CLAUDE → ВОВА_ПРОФІЛЬ).
   - Відповідь у стандартному форматі (звертайся на ім'я).
   - **Чекай «Роби»** — без нього код не чіпати.

Власник визначається автоматично за поштою → `/start` однаково працює і для Роми, і для Вови. Вовин `/startuem` лишається без змін.
